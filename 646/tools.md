# 元の本文
## 生成AI関連ツールレポート

### 画像・動画生成ツール
#### ComfyUI (comfy)
- **概要**: ノードベースの画像・動画生成UI。
- **選定理由・特徴**: 
    - ワークフロー（WF）ベースの運用により、柔軟な制御が可能である点が高く評価されている。
    - RunPodなどのクラウド環境で利用することで、高額なGPUを自前で所有するコストや故障リスクを回避できる。
    - Tensor RTの導入やTiledDiffusionなどのカスタムノードにより、高度なカスタマイズやエラー修正が行われている。
- **主な話題**: 
    - 特定のカスタムノードの不具合や、バージョン不一致による動作不安定に関する議論。
    - LTX2.3系を用いた動画生成ワークフローの構築。
    - 生成速度の向上や、HiDream-O1のネイティブサポートへの期待。

#### Forge (Forge Neo / Forge Couple)
- **概要**: WebUIの高速化・機能拡張版。
- **選定理由・特徴**: 
    - 静止画生成においてはComfyUIを無理に使うよりも十分な性能を持つという意見がある。
    - spectrumとAnima ControlNet LLLiteを併用しても高速に動作する。

#### Stability Matrix
- **概要**: AIツールの一括管理・インストールソフト。
- **選定理由・特徴**: 
    - 複数のAIツールを効率的に管理できるパッケージ管理ツールとして利用されている。
- **主な話題**: 
    - 特定のカスタムノード導入時の制限や、モデルブラウザのフィルタリング機能の仕様変更に関する相談。

#### その他の生成補助ツール
- **anytest**: プロンプトをイメージに誘導する線を描くことで、構図を制御するために利用。
- **DesignDoll**: 3Dデッサン人形として、カメラアングルやポーズ指定の確認に利用。
- **VaM (VirtuAWorld)**: 部位の追従性が高く、容易なポーズ付けが可能なため利用。
- **gptimage**: 単純なマスクインペイントとは異なる挙動による表情差分の作成に利用。

### 学習・タグ付け・管理ツール
#### 学習ツール
- **sd-scripts / kohya-ss**: LoRA学習のデファクトスタンダードとして、最適化手法やキャプションファイルの扱い（信頼と実績があるツール）として利用。
- **Anima-Standalone-Trainer**: 低VRAM環境（RTX 2070 super）でもLoRA作成が可能であるため選定。
- **EmoTion**: 学習設定を自動的に任せられる利便性から利用。

#### タグ付け・管理ツール
- **BooruDatasetTagManager**: LoRA学習用データセットのタグ管理に利用。
- **Taggui (taggui)**: 多くのモデルを内蔵しており、自然言語タグを即座に追加できるメリットがある。
- **pixai tagger gui**: タグ付けの試行に利用。
- **Drawer**: 改修による検索機能の向上とメタデータ閲覧の容易さから利用。
- **Civitai Text Downloader Mod**: Civitaiからのテキスト抽出に利用。

### LLM・対話型AIツール
#### 主要LLM
- **Gemma 4**: 特定のシチュエーションにおける長編テキスト生成や、検閲解除版に知見を読み込ませた「プロンプト職人」としての運用が提案されている。
- **Claude**: 動作しないノードのログを読み込ませてカスタムノードを自作させる手法や、説得によるコンテンツ生成に利用。
- **Grok**: エロプロンプトの通過率が高く、プロンプト作成の相談や文章生成に活用。
- **Gemini**: 原作データの読み込みによるまとめ作成（NotebookLM経由）や、小説執筆における年齢制限の緩さに注目されている。
- **GPT**: 表情差分作成（gptimage）や、年齢認証・規制の挙動についての議論がある。

#### 実行環境・インターフェース
- **LM Studio**: UIが使いやすく、ollamaよりも好んで利用されている。サーバーとして動作させVSCodeと連携させる運用もなされている。
- **SillyTavern**: ロールプレイ中の自動画像生成システムとして利用。API経由でComfyUIワークフローを実行でき、制御しやすい点が高評価。
- **ollama**: ローカルLLM実行環境として利用されているが、UI面に不満が出ている。

### インフラ・プラットフォーム・その他
#### クラウドGPU・ホスティング
- **RunPod**: ComfyUIの動作環境として利用。自前GPUを所有するより合理的であるとされる。
- **Paperspace**: 推論用途において、A6000がほぼ使い放題である点からRunPodと比較検討されている。
- **fal.ai**: HiDream-O1などの最新モデルを試すためのプラットフォームとして利用。

#### 特定用途ツール
- **Irodori-TTS**: Reference Audioによる極めて高い声の再現力から、音声合成に利用。
- **glassworm-hunter**: マルウェア対策として、ソースコード内の不可視Unicode文字等を検知するために利用。
- **Siki**: URL分割の自動修正（置換設定）に利用。
