## 1. 直近窓×再出現スコアが失敗した理由

mod.md / 659 のチャンク別「直近6スレでの再出現スレ数」:

| トピック | rec@r=0.10 | rec@r=0.15 |
|---|---|---|
| 変化の少ないモデル（Anima等） | 1 | 6 |
| Ideogram4 | 1 | 5 |
| seedance2.0 | 0 | 5 |
| Gemma4系 | 0 | 5 |
| Fable5/Mythos5 | 0 | 4 |
| HiDream-E1/O1 | 0 | 0 |
| Hotgen Video V5 | 0 | 0 |

- **r=0.15**: Anima(6) と Ideogram/seedance/Gemma(5) が同程度 → **分離不能**。これらは直近5スレ連続で話題化しており、「直近の再出現」では Anima と区別できない。
- **r=0.10**: ほぼ全員が0〜1 → Anima も漏れる（抑制できない）。

→ 単一半径では「Anima抑制」と「Ideogram保持」を両立する設定が存在しない。

## 3. エンティティ全履歴DF（採用候補）

上表の **総出現スレ数（global document frequency）** はきれいに2群に分かれる:

- **定番（抑制したい）**: anima=61, qwen=86, flux=100, novelai=111, wan=115, illustrious=122 → **すべて ≥ 50**
- **新規・新興（採用したい）**: hidream=40, seedance=14, gemma4=9, ideogram=4, rouwei=4, hotgen=1, plamo=1 → **すべて ≤ 40**

→ **全履歴を通じた出現スレ数 ≈ そのトピックの「定番度」**。Animaは長年の定番なので高DF、Ideogram/seedanceは台頭中なので低DF。直近窓では新興ホット話題も飽和して見えるが、全履歴のDFなら定番だけが突出する（古典的な IDF / ストップワード除去と同じ発想）。

> 注: 全体スレ数は約124（mod）/126（sum）。DBが伸びると母数も伸びるため、**絶対カウント閾値ではなく比率（例: DF/総スレ数 > 0.4 を定番とみなす）** で持つのが望ましい。現データでは anima=0.49 と hidream=0.32 の間にギャップがあり、比率しきい値 ~0.4 で分離可能。

## 4. 意味ベクトル全履歴DFは弱い

mod.md / 659 のチャンク別 全履歴DF（意味検索）:

| トピック | DF@0.10 | DF@0.15 |
|---|---|---|
| 流行モデルの概要 | 11 | 94 |
| Ideogram4 | 1 | 102 |
| seedance2.0 | 2 | 98 |
| Gemma4系 | 2 | 55 |
| Fable5/Mythos5 | 0 | 101 |
| 変化の少ないモデル（Anima） | 15 | 95 |

- **r=0.15 は全件が高DF**（55〜103）。半径が広いと「AIモデルについての話」という一般トピックに過マッチし、特定モデルを識別できない。
- **r=0.10 でようやく分離の兆し**（Anima=15, 概要=11 が高い／新規は0〜2）。ただしこれは「ほぼ同じ文言が過去に何度も書かれたか」を測っているに過ぎず、語彙の揺れに脆い。エンティティDFほど信頼できない。

## 5. 改訂設計の方向性（提案）

検証を踏まえた現実的な設計:

### mod.md（最重要・クリーンに解ける）
- **エンティティ（モデル名）の全履歴 global-DF による定番抑制**を主軸にする。
  - 各チャンクの主モデル（`model` 見出し / `models` メタデータ）について、全履歴での出現スレ数比率を算出。
  - 比率 > `MATURITY_RATIO`（例 0.4）の定番モデル中心のチャンク → 抑制。
  - それ以外（新規・新興・再燃モデル）→ 採用。
  - 複数モデルを含むチャンク（「その他の新モデル」等）は、**含まれるモデルの最小DF**を採用（=新しいモデルが1つでもあれば残す）。
- ほぼ決定論的で説明可能。Anima系は確実に落ち、Ideogram/seedance/Gemma/HiDream/Hotgen/PLaMo は残る。

### sum.md / tools.md（エンティティが明確でない）
クリーンなエンティティが無いため、以下のいずれか:
- **(a) キーワードDF**: 既存の `keywords` メタデータで global-DF を計算し同様に抑制（要・キーワード品質確認）。
- **(b) 逐語的繰り返しの除去**: 意味DF@小半径(~0.10)が高いチャンク（=毎スレ繰り返される定型・概要・基本評価）だけを抑制し、それ以外は残す軽量フィルタ。
- **(c) 現状維持＋微調整**: sum.md は現行LLM凝縮でも要件をある程度満たせている（659で妥当な11件を生成）。過剰設計を避け、mod.md の改善を優先する。

### 共通
- 抑制後の採用チャンクは、既存の `summarize_novel_topics`（LLM凝縮・選別なし）→ 異常時 `format_novel_topics` フォールバックで出力（改訂2の方針を維持）。
- env で閾値調整、`legacy` フォールバックを残し段階導入。
