# 元の本文
**生成AI関連ツール レポート**

### 1. ComfyUI の変化と継続的な進化
ComfyUIは全ラウンドを通じて最も多く言及されるツールだが、バージョン更新や機能追加により状況が変化している点が特徴的である。

- **v0.23→v0.24移行の影響**：v0.23ではNegPipノードやMMAudioノードでエラーが報告されたが、v0.24では全体的に安定動作が確認され、Spectrum Adaptive Forecasterノード追加によりbatch2で生成速度が16秒→12秒程度に向上した事例がある。複数GPU環境ではTurbo併用により11秒台まで短縮された報告もあり、Forge neoを使わずにComfyUI単体で運用する選択肢が現実的になった。
- **ワークフロー面の変化**：rgthree Context BigノードやCoupleノード、外部intノードによるseed固定、JSON構造化プロンプト生成ノードの活用が新たに注目されている。特にIdeogram連携や検閲回避用途でLLM（Codex含む）と組み合わせる事例が増加。
- **選ばれている理由**：ノードベースの細かい制御（Switch、Groupミューター、層別適用など）が可能で、A1111系では設定不要だった機能もノード組み合わせで再現できる柔軟性。Krita連携やStability Matrix経由のデータフォルダ管理も評価されている。

### 2. Forge neo の位置づけ変化
初期ラウンドではAnima生成の選択肢として言及されていたが、ComfyUIの高速化・機能強化により「Forge neoを使わなくてもよい」という声が顕著になった。

- 前日の画像を再生成すると微妙に変化する現象や、バージョン戻しでの再現性問題が報告され、移行検討の要因となっている。
- 後続ラウンドではForge Couple機能のみが部分的に触れられる程度に言及が減少しており、ComfyUIへのシフトが進行中である。

### 3. 新たに登場・注目されたツール
前回・前々回と比較して新たに抽出された、または言及が増えたツールを中心に記述する。

- **Codex（AIコーディング支援）**  
  Anima導入時のエラー対処やComfyUI向けカスタムノード（層別適用ノードなど）の作成で活用。ローカル環境構築やノード作成の手間を大幅に削減できる点が選ばれている理由。

- **RedrayzニキのGUI / trainer_GUI**  
  Anima向け画風LoRA学習で195枚を30分で完了した事例が報告され、プリセット流用や短時間学習のしやすさが評価されている。以前のラウンドではほとんど言及がなく、新たに登場した学習支援ツールとして位置づけられる。

- **ANIMA_Instant_Trainer / Anima-Standalone-Trainer**  
  Colab経由でのLoRA作成ツールとして新たに言及。既存プリセットを活用した実用的な学習が可能な点が理由。

- **Clip Studio Paint（EX版）**  
  AI生成後の漫画制作・加筆修正ツールとして最終ラウンドで新登場。マンガ素材の豊富さと後処理効率の高さが選ばれている理由。

- **Tensorboard**  
  学習時のloss推移を視覚的に確認する用途で登場。学習停止の自動判断の代替として機能する点が評価されている。

### 4. 安定して言及されるツール（変化点のみ）
- **LM Studio**：Anima向けシステムプロンプト作成やComfyUIとのAPI連携で継続的に使われており、ワンポチインストールとプリセット機能の簡便さが理由。ローカルLLM環境の基盤として位置づけは変わっていない。
- **Stability Matrix**：ComfyUIのバージョン管理（Releases/ブランチ切り替え）や出力フォルダのDドライブ移行で引き続き活用。更新挙動の制御が主な選定理由。

以上のように、ComfyUIのバージョンアップとCodex・Redrayz GUIなどの新ツール登場により、生成・学習・後処理のワークフローが全体的に効率化・多様化している傾向が確認できる。
