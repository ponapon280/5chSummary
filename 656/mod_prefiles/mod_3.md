**抽出された生成AIモデル関連話題（主にAnima中心）**
### 1. Anima（WAI-Anima / Anima Base1）に関する言及
- **最も多く話題に上がっているモデル**。
- **選ばれている理由**:
 - 自然言語プロンプトの理解力・応答性が非常に高い（「これですね」と正確に出してくれる）。
 - エロ表現・過激なシチュエーションへの耐性・出力の良さ（検閲が緩いor学習されている）。
 - 複数キャラ同時出しのポーズ・絡み表現がリアスより強い。
 - プロンプトの長文耐性や文脈理解がSDXLより優位。
 - ポン出しでも画風が比較的安定している（特にWAI-Anima）。
- **弱点・注意点として挙げられている点**:
 - 過学習耐性が高すぎて、LoRAの画風乗りが弱い（特にbase）。
 - 複数キャラLoRAを使う場合は「2キャラ写ってる教師画像」を意図的に学習しないとキャラが混ざりやすい。
 - Text EncoderがQwen固定のため、他のTEに変更できない。
### 2. Qwen（AnimaのText Encoder）に関する言及
- AnimaのTEとして使われている。
- **選ばれていない理由**:
 - 日本語が極めて苦手。
- そのため「Qwen以外でTEを使いたい」という要望が複数出ているが、**不可能**（shapeが合わない）と結論づけられている。
### 3. Gemma4（特にGemma4-31B）に関する言及
- **Qwenの代替として選ばれている**。
- **選ばれている理由**:
 - 日本語がQwenより遥かに強い。
 - booruタグも猛虎弁も扱える。
 - 32B規模でもそれなりに賢い。
- 主に「Animaの代替としてクラウドLLM的な用途」や「AnimaのTE以外で日本語を扱いたい場合」に名前が上がる。
### 4. Illustrious（リアス / IL）との比較
- Animaと**最も比較されているモデル**。
- **Animaが優位とされる点**:
 - プロンプト理解力・自然言語制御。
 - キャラの絡み・ポーズの再現性。
- **リアスが優位とされる点**:
 - 画風の扱いやすさ。
 - 既に多くの人がLoRAでコントロールしている実績。
- 結論として「Animaはリアスの上位互換になり得るが、まだ完全に置き換わるほどではない」という意見が複数ある。
### 5. その他のモデル言及（少なめ）
- **WAI base1.0**: Animaの元になったベースとして言及。WAI-Animaとの比較で「WAI-Animaの方がエロ寄り」と評価される。
- **Flux / Qwen-Image**: i2i用途で名前だけ出るが、Animaほど深く議論されていない。
- **Z-Image**: 「もう使わなくなった」と過去のモデルとして言及。
### Web検索による参考情報（2025年時点の主な情報）
- **Anima**: Qwen系をベースにした画像生成モデル（特にQwen2.5-VL系を画像特化でファインチューンしたものとみられる）。2025年に入ってから急に話題になり始めたローカル勢の新星。
- **WAI-Anima**: Animaをベースにさらにエロ寄りに調整した派生版（WAI = Whatever Anime Image?）。
- **Illustrious（リアス）**: 2024〜2025年にかけて人気のアニメ特化モデル。自然言語対応を売りにしているモデル群の一つ。
- **Gemma4**: GoogleのGemmaシリーズの後継。31Bクラスが特に日本語性能で評価されている。
**まとめると**、このログでは**Anima（特にWAI-Anima）**が圧倒的に話題の中心で、「**自然言語理解とエロ耐性**」が選ばれる最大の理由として繰り返し挙げられています。リアスとの比較が非常に多いのも特徴です。

**抽出されたモデル関連話題（除外対象除く）**
- **Gemma4（Gemma4 31B / 12B）**
 主な話題：ローカルLLMとして日本語対応・booruタグ対応が強く、検閲が緩めorなしでクラウドLLMに代わる選択肢として評価されている。31Bはdenseモデルで「小さい割に賢い」「猛虎弁もいける」との声。12Bも試用され「26B-A4Bより良い」との報告あり。
 選ばれている理由：日本語性能・検閲の少なさ・booruタグ理解度。
- **SDXL**
 話題：プロンプト長や自然言語記述への反応性、LoRAとの組み合わせ、構図のランダム性保持でAnima系と使い分けられる点が言及。長い自然言語プロンプトでも比較的安定して反応すると評価。
- **SD3.5**
 話題：比較対象として「クソ重ド無能の代名詞」と酷評され、WAI-Animaなど新モデルとのミーム的な比較で登場。
- **WAI（WAI base / WAI-Anima / waianima）**
 話題：新しくbase1.0版が登場したとされ、ポン出し時の絵柄安定性・エロプロンプトへの反応の良さ・LoRAの効きが話題に。Anima系と比較して「若干ピーキーさが緩和されたbase1」という印象を持たれている。
 選ばれている理由：エロ寄りプロンプトへの強さ・画風の安定。
- **Ideogram 4.0 / Ideogram**
 話題：ComfyUI対応で話題になったが、検閲の厳しさが英語圏でも非難されている。JSONプロンプト対応・日本語対応・2k解像度生成が可能という特徴が挙げられる一方で「検閲さえなければ良かった」との声。
- **その他クラウドモデル（Grok、GPT-image-2、Gemini、Claude、ChatGPT）**
 話題：検閲の有無やエロ表現への対応、ローカルモデルとの比較で言及。Grokは「ドスケベ」と好評、Geminiは最近の改悪（水着すら弾かれる）で不満が出ている。
