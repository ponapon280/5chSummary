# 元の本文
**生成AI関連ツール レポート（変化点・新規登場中心）**

### 1. Wan2GP（新規登場）
ComfyUIを敬遠するユーザー向けの代替ツールとして初めて言及された。  
**選定理由**: ComfyUIのノードベース操作が苦手な人でもKrea2関連のワークフローを比較的簡単に扱える点が評価されている。ComfyUIの複雑さを避けたい層の受け皿として位置づけられている。

### 2. Musubi Tuner（GUI開発の進展）
Krea2 LoRA学習ツールとしての利用が継続しているが、**Gradio版GUIの自作検討・開発**が新たに報告された点が主な変化。  
低VRAM（16GB）環境での実用Tips（`blocks_to_swap 26` + `--block_swap_h2d_only` + `--novram`/`--fp8_base`）も具体的に共有されており、学習速度の改善（20s/it→12s/it）が確認されている。  
**選定理由**: 低VRAM環境での学習可能性と、Krea2との相性の良さ。GUI自作により学習敷居を下げる動きが新たに始まっている。

### 3. ComfyUI関連（セキュリティ・新ノード・機能更新が顕著）
複数回にわたり状況が変化しており、特に以下の点が重点的に更新された。

- **セキュリティリスクの顕在化**  
  カスタムノード導入時に「ハッキングされた」事例が報告され、コード確認の重要性が強く指摘されるようになった。ComfyUI-Managerのブロック機能だけでは不十分で、インストール前にコードレビュー（pycデコンパイル含む）を行う事例が増えている。

- **新ノード・機能の登場・更新**  
  - `ComfyUI-ConditioningKrea2Rebalance`（Krea2向け編集ノード）の追加  
  - `ComfyUI-Krea2-Enhancer`の登場と安全性検証事例  
  - int8ネイティブサポートの強化（ClipLoaderの0.26.0更新必要）  
  - Inspire Pack + prompt controlの組み合わせによるプロンプトスケジューリング機能の代替実現

- **高速化関連**  
  sageattentionの導入事例が増加。生成速度向上（数秒程度）と古いGPUでの効果が確認され、Stability Matrix経由での導入が推奨されている。

**選定理由の傾向**: カスタマイズ性・柔軟性の高さに加え、最近は「コードの安全性確認が可能」「メモリ管理の改善」「速度向上」の観点が強く意識されるようになった。

### 4. Forge neo（新規言及）
古い生成画像の再生成時にModel名を自動記録できる点が評価され、言及された。  
**選定理由**: Model hashからモデルを特定する手間を省き、現在のメタデータ形式に近づけられる利便性。

### 5. その他（変化の小さいツール）
- **Stability Matrix**: 環境構築・カスタムノード導入の容易さは継続的に評価されているが、大きな変化は報告されていない。
- **A1111 / RunPod**: 互換性用途や高性能GPUレンタルとしての言及はあるが、前回からの目立った変化は少ない。

**総括**  
今回特に注目すべきは「Wan2GPの新規登場」「Musubi TunerのGUI開発」「ComfyUIにおけるセキュリティ意識の高まりと新ノード追加」の3点です。ComfyUIは依然として中心的なツールですが、利便性だけでなく安全性や導入ハードルの観点で状況が変化しています。
