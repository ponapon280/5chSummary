**抽出結果**

### 1. Anima（Anima Base / Anima系モデル）
- **主な話題**
  - ControlNet（Tile & Repair ControlNet-LLLite V2.0 / V3.0）の開発が進んでいる。
  - ポーズ系・コンセプトLoRAの開発者が一定数いる。
  - 小物・特徴の学習が非常に強い（「animaは本当に小物とか特徴とかよく学習してくれる」）。
  - エロの柔軟性・表現力が高い（特に2次エロ）。
  - LoRAの相性が独特で、癖が強いという指摘あり。
  - 現在も主力として使われているが、「もう時代遅れになりかけてないか」という声も出始めている。

- **選ばれている理由**
  - エロの出力の強さと柔軟性（検閲が緩い）。
  - キャラクターLoRAや小物・構図の学習精度が高い。
  - コミュニティが成熟しており、LoRA資産が豊富。
  - 低スペックでもある程度扱いやすい。

### 2. Krea2（Krea2 RAW / Krea2 Turbo）
- **主な話題**
  - Animaからの移行先として最も多く語られている新モデル。
  - LoRA学習は **Musubi Tuner** が主流（blocks_to_swap、H2D onlyオプションなどのVRAM対策が話題）。
  - 検閲がAnimaより厳しめで、エロ用途ではまだ物足りないという意見が多数。
  - ただしNSFW LoRAを併用すると、暴力表現などの検閲も一部緩む現象が発見されている。
  - 画風LoRAの相性や背景の強さが評価されている。
  - 生成速度やLoRA作成のしやすさでAnimaを上回る可能性を指摘する声あり。

- **選ばれている理由（または選ばれつつある理由）**
  - 改良スピードが非常に速い。
  - プロンプト追従性・指・衣服の崩れにくさでAnimaを上回る部分がある。
  - LoRA作成時の癖がAnimaより少ない。
  - 将来的にAnimaの上位互換になる可能性を期待されている。

### 3. その他のモデル（言及が少なかったもの）
- **ZIT（Z-Image Turbo）**: 「Krea2の影響でオワコンになりそう」という意見あり。ControlNetのポーズ追随性はまだ評価されている。
- **FLUX**: Animaが「FLUX klein製だったら永住するレベルだった」という悔やみ意見のみ。
- **10Eros**: 動画生成で「一強」と言及されている（画像モデルとは別枠）。

### 4. モデル選択の全体的な傾向
- **Anima**：現在もエロ・柔軟性・学習のしやすさで根強い支持。
- **Krea2**：次世代候補として最も注目されており、特に「画風・構成の安定性」と「開発速度」で評価されているが、エロの強さではまだAnimaに軍配が上がるという意見が優勢。
- ローカル勢の間では「エロの強さ・検閲の緩さ・LoRA資産」がモデル選択の最重要ポイントになっており、この点でKrea2はまだAnimaを完全に超えていないというのが現在のコンセンサス。

**抽出されたモデル関連話題（除外モデルを除く）**

### 1. Krea2（主な話題）
- **主な議論内容**:
  - Krea2のワークフロー（WF）進化：`ComfyUI-ConditioningKrea2Rebalance → Krea2T-Enhancer → Krea2FilterBypass` の順で改良されている。
  - LoRA適用時の挙動：VRAM使用量が急増してOOM（Out of Memory）になる問題が複数報告されており、メモリ管理引数（`--novram` や block swap 設定）で対応する事例が多い。
  - 検閲回避：戦国時代的な状態で、Mystic LoRA（強度0.4推奨）や各種エロLoRAの組み合わせが試されている。text encoderの検閲除去は効果が薄いとされる。
  - 学習・生成：Musubi-tunerを使ってLoRA学習が可能（blocks_to_swap=26 + H2D onlyオプション推奨）。RAWモデルでLoRA学習、turboモデルで生成という公式的な使い分けが言及されている。
  - 画風・性能：背景の再現性が高く、アニメ調への変換やセミリアル（doll系）、ピクセルアート、リアルメガネっ娘などの表現が評価されている。一方でエロの強さは「マイルド」との声もあり、Animaからの移行を検討する声と「まだエロは弱い」との声が混在。
  - その他：Krea2 Image Edit Rebalanceノードの追加、subgraphのunpack必要性、16GB VRAM環境での実用性など。

- **選ばれている理由（言及された点）**:
  - LoRAの癖が少なく学習しやすい。
  - 改良スピードが速い。
  - プロンプト追従性・指の構造・衣服の質感が安定。
  - ローカル環境で無検閲寄りで扱いやすい（クラウド系より表現の自由度が高い）。
  - 背景の強さから、Krea2で下書き→別モデルで仕上げる使い方が提案されている。
