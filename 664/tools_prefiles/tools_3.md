**抽出された「ツール」関連の話題**

- **ComfyUI（comfy）**  
  - 複数回登場し、特に**int8ネイティブサポート**（unet loader / clip loader）が話題。convrot対応の有無やtensorwiseサポートについても言及。  
  - カスタムノードの活用（BREAK対応、RNG:GPU再現など）が多く、**柔軟なワークフロー構築**や最新機能の取り込みを理由に選ばれている。  
  - 0.26版の評判や、gguf/int8対応の進化が評価されている。

- **A1111 / WebUI（Automatic1111系、Forge Neo含む）**  
  - 古い生成画像の再現用途で言及。**RNGをGPUにする**ための互換環境として利用。  
  - ComfyUIとの併用（環境を分けて古いメタデータを扱う）で選ばれるケースあり。smZNodesとの相性問題も話題。

- **Stability Matrix（SM）**  
  - バージョン管理・環境構築ツールとして登場。  
  - 更新後の不具合（i2iエラー）で旧バージョンに戻す使い方が確認され、**複数環境の管理のしやすさ**が利点として挙げられている。

- **musubi tuner**  
  - 学習（特にAnima向け）のテスト用途で言及。  
  - 試行回数が増えるため設定検証が大変になった点が指摘されており、**学習速度・柔軟性**を評価して使われている。

- **その他のComfyUI関連ツール・ノード**  
  - sageattention（2.2.0.post5、dim256対応whl）  
  - 各種カスタムノード（smZNodes、NAGなど）の導入・置き換え議論  
  - これらは**省VRAM・高速化**や旧環境再現を目的に選ばれている。

**補足**  
- Krea2、Anima、Qwenシリーズ、Z-Imageなどはすべて**モデル**として扱われているため除外。  
- QwenはKrea2のベースとして出てくるが、画像生成用途のためツールとしては抽出対象外。  
- ツール選択理由として共通するのは「**VRAM制約下での動作性**」「**旧環境・旧画像との互換性**」「**カスタム機能の拡張性**」の3点。
