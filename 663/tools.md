# 元の本文
**生成AI関連ツール レポート**

### ComfyUI（主な変化点・重点記述）
前回・前々回と比べて最も状況が変化しているのがComfyUIです。VRAM制限を超えた大規模モデルの扱いが可能になった点が新たに評価されており、「comfyUIがVRAMを越えたモデルを扱えるようになったのは大きい」との声が複数見られます。これにより、Krea2のNVFP4版（7.1GB）のような軽量・大規模モデルをローカルで安定して動かせるようになったことが、選ばれる主な理由となっています。

また、更新後のノード互換性問題（CLIPNegPipノードのエラー発生）や、bbox機能によるキャラ位置・構図の細かい制御が可能になった点も新たに言及されています。refiner＋Ultimate SD UpscaleやTiled Diffusionを組み合わせた領域分けワークフローの実用性が高まっており、「Krea2のbbox制御を活かしたい」「複雑な構図制御がしやすい」といった理由で選択されています。

公式テンプレートの拡充や、LLMにワークフローを組んでもらう運用が一般化している点も、前回までと異なる変化です。省メモリ・高速化や「決まりきった手順の自動化」を目的とした導入が増えています。

### LoRA学習ツール（新たに登場・注目）
musubi-tunerとai-toolkitが新たに具体的に言及されるようになりました。特にKrea2向けLoRA作成で利用されており、VRAM16〜24GB環境でも実用可能な点が評価されています。

- **musubi-tuner**: `--blocks_to_swap 25`などの省メモリオプションを組み合わせ、1024解像度で2000ステップを9時間程度で回す事例が報告されています。
- **ai-toolkit**: VRAM24GB前後で3000stepsを3時間弱で処理する報告があり、低〜中スペック環境での実用性が選ばれる理由となっています。

これらは前回までほとんど登場していなかったツール群で、Krea2対応の進展に伴い新たに注目されています。

### その他の変化
- **StabilityMatrix**: ComfyUIのインストール・管理ツールとして新たに言及。環境構築のハードルを下げる役割で選ばれています。
- **webUI / Forge系**: 移行話は依然としてありますが、最新モデル対応やVRAM管理の観点でComfyUIへのシフトがさらに進んでおり、操作の簡単さという利点以外での言及が減少傾向にあります。

以上のように、ComfyUIの機能強化と、それに伴うLoRA学習ツールの新登場が今回の主な変化点です。
