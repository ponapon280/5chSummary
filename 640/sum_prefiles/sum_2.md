# なんJ AI画像生成スレッド レポート（2024年4月頃推定）

## 概要
このログは、5chなんJスタイルのAI画像生成スレッド（主にStable Diffusion XL (SDXL) 関連）で、**GPT-Image-2**の新機能登場と**Anima**（高性能SDXL派生モデル）の最適化・活用が大半を占める。参加者（通称「ニキ」）は生成クオリティの高さ（「ゲームエンド」級）を絶賛しつつ、エロ制限・規制強化の懸念、Detailer（顔/手修正ツール）の不具合解決、LoRA学習Tips、高速化手法（Turbo LoRA, Spectrumなど）を熱く議論。ハードウェア（RTX 5090/4070/3060など）の生成速度比較も活発。全体230〜441番台で、GPT-Image-2の衝撃がAnimaの盛り上がりを後押し。エロ生成（プッシー/オマンコ描写）の限界や実在人物生成のリスクが繰り返し指摘される。

総勢100+レス中、**GPT-Image-2: 約40%**、**Anima最適化: 約50%**、その他（規制/LoRA/生成例: 約10%）。

## 主要トピックまとめ

### 1. GPT-Image-2の衝撃と懸念（>>230〜>>441のピーク）
- **絶賛ポイント**:
  - プロンプト理解力抜群。「昭和アイドルのDVDパッケージ」→松田聖子風生成（>>273）。ギャルゲー紹介文画像（>>393: アマガミ感ヤバい、予約特典まで再現）。
  - オリキャラ設定渡しでデザイン生成（>>376, >>358）。淫語ロゴ複数案ポン出し（>>274, >>287, >>291）。
  - 構成力高く、サイズ感おかしい依頼も破綻なし（>>418）。マニア向け記事（フラッシュライトレビュー）完璧（>>403）。
- **問題点**:
  - エロ厳格: 水着/ビキニOKだが「ドスケベオマンコ」拒否（>>275, >>334）。17歳設定でウェディングドレスNG（>>309）。
  - 実在人物/商標自動生成: 松田聖子、企業ロゴ（Rマーク）出る（>>273, >>295）。架空指定必須（>>300）。
  - 規制予測: Sora2ルート（最初無規制→ナーフ）再来（>>247, >>292, >>302）。pornhub判例適用懸念（>>242）。
- **比較**: Anima/bananaより同一性上、着せ替え/無から生成強いが、ディテール劣る（>>298）。フォトリアルは感動薄（>>275）。

### 2. Animaの最適化・活用（高速化/Detailer中心）
- **高速化Tips**:
  | 手法 | 速度例 (RTX5090/4070) | クオリティ影響 | 参考レス |
  |------|-----------------------|---------------|----------|
  | Turbo LoRA (CFG1, Step8-12, 1024x1024) | 0.9-1.3秒 | 劣化少、ガチャ向き。プロンプト効き/構図犠牲 | >>278, >>336, >>352 |
  | Spectrum Adaptive Forecaster | +20-30%速 | 構図ほぼ変わらず細部/色味向上。Turbo共存可 | >>328, >>348, >>375 |
  | DMD2 | 代替可だがクオリティ落ち | Turboよりメリット薄 | >>308, >>380 |
  | SageAttention + TorchCompile | 通常時4-5秒→大幅短縮 | 推奨 | >>321, >>353 |
  | Highres LoRA + 解像度落とし (512x512 + Upscale) | ロープロ対応 | 構図安定 | >>320 |

  - 5090で素30秒→Turbo+Spectrumで1秒台（>>336, >>352）。3060で160秒かかる報告も（>>384）。
  - Mac DrawThings: Anima 8bitSでNPU活用、1.5-2倍速（>>370）。

- **Detailer問題解決** (>>238起点):
  - 症状: 顔/手暗く/網状汚れ（shaded face誤認識、LoRA未適用）。
  - 解決策: Denoise低め(0.4)+Step10+CFG4、モデル直結（Detailer前Upscaleなし）、ライティングプロンプト追加、影除去（>>258, >>260, >>373, >>397, >>411）。手動修正推奨（>>384）。
  - Anima素でも瞳綺麗、Detailer不要派（>>266）。

- **LoRA学習Tips**:
  - バッチ3まで（4+でマスピ顔化）（>>402）。品質/絵師タグ追加で再現向上（>>371, >>409）。
  - タグ過多で悪化（1980s styleなど）（>>401, >>404）。自然言語キャプション: Kohya image caption CLI（>>412, >>426）。
  - キャラ再現難: x-shaped pupils溶け（>>377）、紐安定死（>>410）。プロンプトでスタイル指定（>>362）。

## 結論・示唆
スレはGPT-Image-2の「プロンプト→完璧構成」力で沸騰中だが、Animaの実務活用（高速/LoRA）が本丸。エロ/権利制限が足枷で、規制ナーフ不可避と悲観。ニキのTips共有文化が強く、初心者ハードル低。次スレはAnima正式版/規制後対応か。生成速度向上で「1秒ガチャ」時代到来中。
