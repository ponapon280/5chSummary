**抽出された生成AI「モデル」関連話題（除外リスト適用済み）**

除外対象（NovelAI/NAI、illustrious/リアス/ill/IL、FLUX、Wan、Qwen-Image、anima、Z-Image系、LTX系）の言及はすべて無視。該当する話題のみを抽出し、特に「なぜそのモデルが選ばれているか／強み・用途」の記述を優先してまとめた。

### 1. Krea2 / Krea / krea2（実写・背景特化）
- **主な言及箇所**: 436, 437, 454, 539, 558, 559, 562, 575
- **選ばれている理由・評価**:
  - アニメ絵はAnimaだが「実写ではKrea2が猛威振るってる」。背景が特に強い。
  - キャラをAnimaでアニメ風、背景をKreaで写実風に組み合わせる使い方が流行（「ワイブーム」）。
  - int4conv量子化時の劣化がAnimaより少なく、出てくる画像はまだまとも。完全4bitだと劣化は目立つが、一部8bit混在でサイズと品質のバランスが取れる。
  - 低スペ（4060など）でもint4convなら実用可能という期待。
  - ケチケチワークフローで生成時間を大幅短縮できるが、やりすぎると品質が落ちる例あり。
- **文脈**: 量子化（int4/int8 convrot）の実験対象として頻繁に登場。実写・背景用途でAnimaと明確に役割分担されている。

### 2. Grok（画像生成・画像取り込み・エージェント）
- **主な言及箇所**: 439, 445, 516, 528, 532
- **選ばれている理由・評価**:
  - 元絵にモザイク処理があればエロワードを吹き出しやロゴに変換しやすい（後で「実写はモデられた」と訂正あり）。
  - 以前出せなかった表現（ボテ腹乳首、男女抱き合い、ペニスなど）が改善されつつある。生成はするが表示可否は別、という挙動。
  - 画像取り込み機能が進化し、積極的に出せるようになっている（規制も早そうという警戒あり）。
- **文脈**: エロ寄り画像生成や編集用途。専用スレ誘導も出るほど話題。

### 3. GPT系（GPT Live / ライブGPT / チャッピー / ChatGPT / GPT5.6 / Ember）
- **主な言及箇所**: 442, 443, 446, 447, 455, 465, 506, 601, 628
- **選ばれている理由・評価**:
  - **GPT Live（ライブGPT）**: 「自然すぎ」「ほんとに会話してる気分になる」。直球エロだけでなく壁打ち・雑談にも高性能で使えそう。喋りがやや外人っぽいが問題にならないレベル。
  - チャッピーPlusユーザーでも「自然な話し方のモデルが無い」（全部フィリピンママっぽい）という不満。Emberというモデル名が候補として挙がる。
  - GPT5.6：カスタム指示（お嬢様GPTなど）のアップデートに使用。やり取り自体が楽しい。
  - 全体として「チャッピー廃人が量産されそう」なほど会話性能が高い。
- **文脈**: テキスト対話・音声ライブチャット用途。画像生成より会話の自然さが強調される。

### 4. Claude
- **主な言及箇所**: 470, 551, 558
- **選ばれている理由・評価**:

  - adaroundなどの技術的相談相手としても機能。
- **文脈**: 量子化・ComfyUIワークフロー改善のアシスタントとして非常に頼りにされている。コード・設定解析に強い。
