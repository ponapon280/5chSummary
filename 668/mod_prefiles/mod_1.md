**抽出結果（該当モデルのみ／ログ内の関連話題と選択理由）**

### NovelAI (NAI)
- 話題：NAIの話題が少ないことへの言及（FAQに「Q.NovelAIの話題が少ないんだけど」を追加した）。最近NAIに課金し始めたがスレで誰も話していないという残念の声。

### illustrious（イラストリアス／リアス／ill／IL）
- 話題：Animaとの比較で頻繁に登場。チェックリストでも「Illustrious系からプロンプトを流用していない」が挙げられている。
- 選択／比較理由（Anima優位として）：
  - リアスだと過学習させてLBWを使うなど手間がかかるが、Animaは良い。
  - エロ漫画1ページのLoRAで手マンを綺麗に再現できるが、リアスでは無理だった。
  - リアスからのプロンプト流用は避けるべき（Anima質問時チェックリスト）。

### Wan
- 話題：wan2.2のモデルとTEをINT8 ConvRot化した話。RTX 5090で1生成約130秒→ほぼ半分に高速化して感動。Wan2.2のチューンモデル（SmoothMix / DaSiWa / EXITIUM VICTRIXなど）のfp16/bf16版の有無やint8化の話。Wan2.2のようなCoEではなくMoEっぽいので速い可能性への期待（長尺動画関連）。
- 選択／使用理由：INT8化による大幅な高速化が実感できるため。

### Qwen-Image（ログ内ではQIE／Qwen関連）
- 話題：自然言語をjsonに強化するリライターがQwen3.6-27Bであること。昨日「QIEのピクセルシフトを完全に防ぐ生成方法」が出てきた（Xで話題、kohyaも驚いた）。
- 選択／使用理由：ピクセルシフト完全防止手法が登場した点で注目。リライター用途での言及。

### anima（Anima／アニマ）
- 話題が最多。base / Aesthetic / Aesthetic-b / turbo / turbo LoRA / int8化 / 亜種乱立 / LoRA適性 / チェックリストなど多岐にわたる。
- 主な話題と選択理由：
  - 軽くて扱いやすいので、色んな合体・調教・マージがされやすい。
  - リアスより良い（過学習＋LBW不要寄り、LoRAで手マンが綺麗に再現できる）。
  - 公式から色々出たが結局baseでよさそう。
  - Aesthetic＋turboはおしっこがちゃんと尿道から出る確率が高い（base＋turboより優秀）。Aesthetic-bの方が良い（1.0はプロンプト追従が下がる印象。bは美的微調整のみでスタイル調整・安定化LoRAなし）。
  - turbo v1は絵柄が安定する（デフォルト絵柄安定寄り）。turbo＋turbo LoRA（ツインターボ）はエロ文字クオリティが高く、汗の勝手描きなどが良い方向に効く場合あり。int8化＋turbo LoRAで非常に速いが構図固定化・細部簡略化に注意。
  - LoRAに影響されやすい（「SDXL的なええ感じに混ぜとく」がないので特定ポーズ・性癖が出せなくなる場合あり）。覚えが良いのでゴミ画像で枚数稼ぎは悪手。
  - krea2のLoRA学習と比べて同ステップで3〜4倍速い（時間効率良い）。
  - AnimaとKrea2が出揃ってNAI/SDXLを過去にした。
  - 質問時チェックリスト多数（Turbo/TurboLoRA/INT8を使っていない、Illustrious系プロンプト流用なし、矛盾プロンプトなし、BREAKなしなど）。int8/turboは構図固定化・簡略化を避けられないので注意喚起。
  - 亜種がたくさん出る（baseが順当に上がると思ったら亜種乱立）。

### 該当なし
- **FLUX**：言及なし
- **Z-Image（Z-Image Turbo / ZIT / ZIE）**：言及なし
- **LTX（LTX-2.3 / ltx2.3）**：言及なし

**補足**：ログ全体でAnimaが圧倒的に多く、比較対象としてリアス・Krea2・NAIが絡む形。選択理由は主に「軽さ・扱いやすさ・LoRA適性・特定表現の精度（手マン・尿道など）・速度（turbo/int8）・baseの安定感」が中心。Wanは高速化（INT8）、Qwen関連は特定手法・リライター用途で触れられている。

**抽出結果：生成AI「モデル」に関する話題（除外リスト適用済み）**

除外モデル（NovelAI/NAI、illustrious/リアス/ill/IL、FLUX、Wan、Qwen-Image、anima、Z-Image系、LTX系）およびそれらに直接紐づく話題（aesthetic/turbo派生を含むAnima関連、Wan2.2/int8化、QIEなど）はすべて除外。該当するモデル話題を以下に整理。特に「選ばれている理由」があるものを優先・強調。

### 1. Krea / Krea2 / クレア
- 低スペでも早くてエロいKrea2ワークフローを求める声（>>22）。
- 「結局クレアの覇権は変わらなそう」（>>27）。他モデルを「クレアのいない時代に生まれただけの凡夫」と位置づけ、依然として主流・優位と認識。
- Kreaの性器表現が「グ◯マンになりがち」との指摘（>>69）。
- Krea2で実写ハーレムやイチャイチャカップルを生成すると、ネトラレ物を読んだ時のような感覚が来て「脳破壊されそう」になった体験談（>>106）。
- Krea2のLoRA学習が時間かかる（Anima比較で同ステップ3〜4倍、3090+musubiで高速化オプション全開でも重い）との実践報告（>>117）。

**選ばれている主な理由**：低スペでも高速・エロい生成が可能、現状の覇権モデルとしての安定感・優位性、実写寄りエロ（ハーレム/カップル）での没入感。

### 2. Grok（Grok4.5含む）
- クラウドAIでエロ生成すると逮捕リスクありとの警告。Grokで継娘の性的画像を大量作成した男性が自殺した事例、SpaceXAIが警察協力を拒否し捜査妨害したとの報道共有（>>30）。
- 実在児童ポルノ生成＋共有で捕まる事例がある事実確認（>>32, >>37）。Grokで実写系エロをやると実際に捕まるリスクあり。
- 「エロ関係やるならGrok使え」（>>55）。
- 安くてエロ緩いfastモデルをまた出してほしい。5ch要約に最適だったとの評価（>>47）。
- grok4.5のエロ小説がかなり良い（>>155）。
- Seedance、Seedreamとともに「Grokの新しいのが来とる」が、ローカルエロ画像生成を優先して試していない（>>164）。

**選ばれている主な理由**：エロ規制が緩い、エロ小説生成の質が高い、安価・高速モデルとして実用的（要約用途含む）。一方で実写エロ/CSAMリスクが強く指摘され、クラウド利用への警戒がセットで語られる。

### 3. OpenAI / GPT系（GPT-5.5、GPT-5.6、Terra、Luna、ChatGPT、GPT-Image）
- OpenAIが最新「GPT-5.6」を一般公開予定（日本時間10日頃）。Terraは日常業務向けバランス型でGPT-5.5相当性能・価格1/2、Lunaは高速・コスト効率型（>>39, >>43）。
- OpenAIは軽量安価モデルをよく出してくれる（>>47）。
- ChatGPT Plusで5.5高思考バージョン使用中に、エロ混じりのワークフロー相談で回答が二転三転して「いきなりアホになってない？」との報告。エロ関連が原因かと疑う声（>>54）。
- GPTもGeminiもバージョンアップ前は挙動がおかしくなるのが恒例なので待つべき（>>58）。
- GPT-Imageで教師画像（武器など構造がはっきりしたもの）を作成する提案（>>101）。
- GPT5.6の性能を試したい（>>128, >>130）。ただしスレ民には5.5ですでにオーバースペック。エロ規制が緩むなら期待だがOpenAIに限ってそれはない（>>131）。
- サムのエロ解禁発言が何度も延期→公式に開発断念発表済みなのに「まだ続く」と思っている人がいるとの指摘。エロチャレンジBANの対応を批判（>>133, >>138）。
- 動画生成の参照用絵コンテをGPT-Imageで作る用途でXで活発（>>223）。

**選ばれている主な理由**：新バージョンの性能・価格バランス（Terraの半額相当、Lunaの高速安価）、軽量モデルの継続リリース、GPT-Imageの教師画像/絵コンテ生成実用性。一方でエロ規制が極めて厳しく、エロ混じりプロンプトで挙動が不安定になる、エロ解禁は公式断念済みという否定的評価が強い。スレ民にはすでにオーバースペックとの声も。

### 4. その他のモデル言及
- **Gemini**：GPTと同様、バージョンアップ前に挙動がおかしくなるのが恒例（>>58）。
- **Seedance / Seedream**：Grok新版とともに「新しいのが来とる」が、ローカルエロ画像生成を優先して未試用（>>164）。
- **deepseek v4 pro**：なんJNVA部に関する知識テストで使用。特定の回答パターンを示す例として言及（>>213）。
- **Animagine**：AnimaではなくAnimagineを使っているケースがあるとの言及（>>143。除外モデルとの対比のみ）。
- **addift**：画像ペア1組だけで学習できる手法として「気軽に学習できていい」「この手法思いついた人すごい」（>>163, >>173）。モデルそのものではなく学習手法だが、生成関連として軽く触れる。

### 全体的な傾向・補足
- クラウドモデル（Grok、GPT系）はエロ用途での「緩さ」や安価・高速性が評価される一方、逮捕リスク・規制・挙動不安定が強く警戒される。
- ローカル寄りではKrea2が「覇権」「低スペでもエロい高速」としてポジティブに扱われる。
- 新モデル発表（GPT-5.6系、Grok新版、Seedance/Seedream）は速報的に触れられるが、スレの主戦場はローカルエロ画像生成寄りで、クラウドは補助/小説/ワークフロー用途。
- 除外モデルが圧倒的に多いログのため、抽出対象は相対的に少なめ。特に「理由」が明確なのはKrea2の覇権・速度・エロ適性、Grokのエロ緩さ・小説質、OpenAIの価格/速度バランスとエロ規制の厳しさ。
