### ComfyUI（comfy / comfyui）関連
- **453**: GPUコアを使う作業をブラウザでやると影響が出る。comfyのUIが全然更新されなかった頃と比べるとだいぶ重くなった。
- **467**: （ブラウザウィンドウサイズによる生成速度差の話に対し）comfyuiでも起こるらしい。
- **470**: 昆布（モデル）をclaudeに作ってもらった変換ツールで変換したものを使用。comfyuiが公開している昆布モデルと同じ設定でDiTだけ量子化。それ以外も量子化するとさらに高速化する可能性はあるが品質劣化する、とclaudeに言われた。comfy convモデルのメタデータをclaudeに分析させて設定をパクった。
- **474**: comfy公式が選んだレイヤーをただint8 convにするだけでも高精度。comfy公式が丸め学習をやっている可能性もある。
- **475**: int4 convrot w4a4 のネイティブ対応以外にも **ComfyUI-INT4-Fast** やらなんやらで活況。ComfyUI-INT4-Fastのほうがええんやろか？
- **486**: int4conv対応がいつの間にかcomfyuiにマージされていた。ただint8みたいに調整なしでただ変換するだけだと精度落ちる（PR作者談）。ベストは場所によって4bit/8bit/16bitを切り替えたもの。
- **610**: モデルのサイズやら速度改善はええが、**comfy標準で画像にWF埋め込むときのメタデータも圧縮してくれ**んかな。

### Forge Neo / EasyForgeNeo / forgeneo（WebUI系フォーク）関連
- **441**: EasyForgeNeoでAnimaを使っている。ブラウザの全画面表示・ウィンドウを小さくする・最小化で生成速度差がわからん（1分以上かかる生成で1秒未満）。何年か前のGPUスケジューリング設定変更高速化の話も差が分からなかった。Win10 + 2080Ti環境。
- **450**: WQHD forgeneo + firefox + rx9070 + リアス初期設定で速度計測。1024×1024 100step で最大化22.8sec / 最小化22.6sec。2048×2048 50step で最大化1min1.8sec / 最小化1min1.5sec。hires2倍でも差は0.5secほどで誤差レベル。8Kモニタなら変わるかも？
- **458**: 話題になっているのはForge Neoだぞ（comfyの話に対する訂正）。
- **462-463**: 「今forge neoなんて使ってるやつマジでどういう層？？」「forge neoを使ってる層」というやり取り。
- **理由・文脈**: ブラウザウィンドウサイズ（最大化/最小化/全画面）やLive Previewsの影響で生成速度が変わるかどうかの検証が複数投稿で行われている。差がほぼ誤差か環境依存、という結論寄り。comfyよりForge Neoが話題の中心になっている場面あり。
