# なんJ(5ch) AI生成スレッド ログレポート (レス433-636)

## 概要
このログは、主にComfyUIを活用したAI動画生成（特にSVIモデル）を中心とした議論。エロティックなコンテンツ（百合、射精、拘束など）の生成Tips、ワークフロー共有、LoRA互換性、アップローダー問題が活発。静止画生成（Qwen Image, Z-image, SDXL）の話題も交えつつ、ハードウェア限界や新モデル期待が散見される。全体的に実践者同士のトラブルシュートと共有が主で、2024年末～2025年初頭のトレンド（SVI Pro, Native WF, VRAM16GB運用）を反映。脱線（勃起薬、X凍結）も一部あり。

## 主なトピック別まとめ

### 1. SVI動画生成の核心議論 (最多: 約40%)
- **モデル・LoRA互換性**: 
  - NativeモードではKijai版LoRA（例: `SVI_v2_PRO_Wan2.2-I2V-A14B_LOW_lora_rank_128_fp16`）必須。公式LoRAはwrapper専用でnative不可（>>434,436,449,455）。
  - Pro版は顔変化目立つがプロンプト追従悪化（>>458）。無印版推奨者多数。
- **ワークフロー (WF) Tips**:
  - Native WFはシンプル（getset/サブグラフ除く）。KJNodes git pullで解決（>>434）。
  - PainterLongVideo + SVI Low/High構成で長動画（>>467,472,559,577）。5秒×3で14秒生成、重なり5Fオーバーラップ推奨（>>577）。
  - エンドフレーム指定: PRマージで可能だが終端4F色ズレ発生。色補正/カット必須（>>470,595,599,600,602,628）。
  - ループ: スタート/エンド同画像で制約強すぎ（>>496,499）。専用WF共有（>>509,519,526,577）。
  - SAM3: 自動目Detailer便利。必須ではないが推奨（>>446,451,532）。
  - TripleKSampler: High/Low高速化補完用（>>618,619）。
- **生成Tips**:
  - 色変化: LoRA強度0.8緩和、シードガチャ（>>444,561,607）。
  - 顔安定: アンカー画像強力だがプロンプト無視多発（>>467）。SVI > PainterLongVideo（>>585）。
  - 失敗例: ノイズ/赤ちゃん悲鳴（>>436,440）。5秒短すぎ感（>>581）。
- **共有WF例**: KJNodes nightly必須（>>592）。サブグラフ展開版（>>577）。Native変換版（>>559,623）。

| WF共有元 | 特徴 | 問題点 |
|----------|------|--------|
| >>461,526 | 10秒動画、PLVデビュー向け | litter重 |
| >>509,519 | ループ用、Bypass活用 | tadaupメタ消滅 |
| >>577 | 5秒×3=14秒、プロンプト追従（手を振る/胸揉む） | エンドフレームなし |

### 2. アップローダー・共有問題 (約15%)
- catbox/litter死/重（>>448,457,510）。tadaup代替だがメタデータ（WF/pngコメント）消滅（>>507,514,516,545）。
  - 解決: ibb推奨（>>518）、png EXIF残るが時間経過で消（>>516）。R18チェック意味薄（>>491,492）。
- WF読み込み失敗: 英語環境確認（>>433）、ブラウザ/キャッシュ環（>>512,513）。

### 3. LoRA/モデル関連 (約20%)
- **エロ特化LoRA**:
  - 射精/潮吹き/放尿: Smooth MIX Animation系推奨（>>634）。t2v飲尿LoRAマシだが外人多（>>542）。
  - Qwen Image Edit 2511 NSFW: 百合2キャラLoRA公開（512x512, dim48）。VRAMきつ（>>597,598）。
  - CunnyFunky: MiaoMiao超え、LoRAで乳首溶け対策（>>543,569,573,635）。X凍結騒動（>>546,548）。
- **新/他モデル**:
  | モデル | 評価 | 課題 |
  |--------|------|------|
  | Z-image | Turbo鮮明だがLoRAでSDXL並（>>571）。Base待望（>>524,528）。16GBエロFT期待（>>480,580）。 |
  | Qwen Image Edit 2511/2512 | NSFW LoRA公開、自然言語強（>>500,597）。resolution steps=ピクセル倍数指定（>>588,601）。 |
  | Wan2.2/2.6/Painter | 動き良/色ズレ対策（>>606-612）。 |
  | Flux2/NewBie | VRAM重/生成弱（>>477-479,575）。 |
- セミリアルLoRA: 装飾吹飛ぶ（>>441）。

### 4. ツール/ハード/その他Tips (約15%)
- **ツール**:
  - Lossless Scaling: フレーム補完可だが残像/ティアリング（>>445,452）。
  - SAM3/YOLO: 目/segDetailer（>>451,533）。
  - StabilityMatrix: Wan2GP追加、Easy代替（>>535）。
  - ffmpeg: ストレージ圧縮（>>621）。
- **ハード限界**: 16GB VRAMでSDXL/Qwen運用（>>484,597）。5090待ち（>>487）。
- **エログッズ脱線**: シアリス/ダポキ/ケーゲル（>>462-466,476）。
- **生成悩み**: 透明人間描写難（>>443,447）。see-through対策（>>537）。Wildcardネスト限界（>>564）。

### 5. トレンド・未来展望 (約10%)

## 結論・注目点
- **ホットトピック**: SVI Native WFの最適化（Pro移行中）。エンドフレーム/色ズレ対策が鍵。
- **ユーザー傾向**: 低スペ（16GB）エロ動画実践者多。WF共有文化活発だがアップローダー不安定。
- **リスク**: LoRA削除/CivitaiBAN（>>539）。メタデータ共有工夫必須。
- **次スレ予感**: Z-image Baseリリースで移行加速？ Qwen Edit LoRA拡散。
