**抽出結果**

### 主な話題と選定理由

**1. Anima（Anima base / Anima Turbo）**
- 最も多く話題に上がっているモデル。
- **選ばれている理由**:
  - INT8 ConvRot量子化により、生成速度が大幅に向上（特にRTX 30/40/50シリーズで顕著）。
  - VRAM使用量を抑えられる（例: 8.3GB → 6.2GB）。
  - Anima Turbo LoRAと組み合わせることで、6〜12ステップ程度の高速生成が可能。
  - ローカル環境で常用しやすい軽量モデルとして評価。
- **注意点**: 通常のINT8（非ConvRot）は精度が落ちやすいため、**ConvRot版**を推奨する声が多い。ComfyUI nativeノード対応が進んでいる。

**2. Krea2（krea2_turbo）**
- Animaと並んで活発に比較されている。
- **選ばれている理由**:
  - Animaより細部描写・クオリティが高いという評価。
  - INT8 ConvRot版を使うことで、FP8版より明らかに高速（4070TiSで18秒台 → 14秒台）。
  - LoRA学習・適用がしやすい。
- **弱点として挙げられた点**: VRAM16GBでもFP8/INT8 ConvRotともに載らないケースあり。bbox（バウンディングボックス）制御が効きにくい。

**3. Illustrious（イラストリアス / リアス / IL）**
- SDXL系モデルとして言及。
- **選ばれている理由**:
  - FP8量子化との相性が良く、以前から高速化手法が確立されている。
  - 今でも一部ユーザーが常用。
- 量子化する場合はFP8推奨（INT8 ConvRotはあまり話題になっていない）。

**4. NovelAI（NAI）**
- V4/V5の話題。
- **言及内容**: NAI V5のトレーニングに時間がかかっている（9月頃リリース？）という情報。性能より版権キャラ数の更新を望む声あり。
- ローカル勢はAnimaやKrea2に移行している人が多い印象。

**5. LTX（LTX-2.3）**
- 動画生成モデルとして1回のみ言及。
- int8量子化時の精度低下（静止画化、口パクしない）が指摘されており、**精度面で信用していない**という意見。

**その他のモデル**
- FLUX、Wan、Qwen-Image、Z-Imageについてはこのログ内では一切言及なし。

**量子化関連の共通トレンド**
- **INT8 ConvRot** が現在の主流話題。
- 目的: 速度向上＋VRAM節約。
- 通常のINT8やRow-wiseよりConvRotの方が精度が良いとされ、特に30xxシリーズで効果が高い。
- ただし「品質を最優先するなら素のbf16/fp16のままが良い」という慎重意見も存在。

**Krea2（Krea2 Turbo）**
- ログ内で最も多く言及されている非除外モデル。
- 主な話題：INT8 ConvRot量子化版とFP8版の速度・品質比較。
- 具体例：
 - 4070TiS環境で`krea2_turbo_int8_convrot.safetensors`がFP8版より明らかに高速（15秒台→14秒台）と評価。
 - VRAM16GBでもINT8 ConvRot版が実用的という報告あり。
- 選ばれている理由：**速度向上**（特に30/40シリーズでの生成時間短縮）と「品質劣化を感じにくい」という点が繰り返し挙げられている。
**INT8 ConvRot（量子化手法）**
- モデル自体ではなく、拡散モデルの量子化方式として独立して話題化。
- INT8 Rowwise / Tensor-wise / 素のINT8と明確に区別され、「ConvRotが精度・速度のバランスが良い」との評価が主流。
- ComfyUI nativeノード対応やカスタムノード（int8-fast）との組み合わせが議論されている。
- 選ばれている理由：**生成速度の向上**と**精度の維持**の両立。特にRTX 30/40/50シリーズでの実用性が高いとされる。
