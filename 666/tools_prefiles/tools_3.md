**抽出結果（ツール関連のみ）**

### ComfyUI（comfy）関連
- **言及内容**:
  - ComfyUIのラッパー作成（複数モデルのマージを簡単にするUI追加、マージ時のメモリ枯渇対策として複数回に分けて処理する機能を実装）。
  - ワークフロー（WF）の作成・改良が活発（i2i用にLoad Image + VAE Encodeを追加する基本構成、部分的i2iにSet Latent Noise Maskを使用、ControlNet対応WF、Depth Processor用WF）。
  - カスタムノードの活用（KijaiノードのDepth Processor、ConvRot関連ノード、AnyTest→Inpaintの組み合わせ）。
  - ブラウザスクリプトだけでなく、ComfyUI自体で様々な機能を作成したいという声。

- **ツールが選ばれている理由**:
  - ワークフローの柔軟性・カスタマイズ性が高い（i2i、ControlNet、部分的編集、マージ処理など自由度が高い）。
  - 速度最適化ノード（ConvRot INT8など）の組み込みが容易で、高速化が期待できる。
  - 複雑な処理（複数モデルマージ、i2i、ControlNet）をノードベースで視覚的に構築できる。

### Forge関連
- **言及内容**:
  - PC内に無印のForgeが入っているが、アップデートがないため使用頻度が低いという話。

- **ツールが選ばれている理由**:
  - 特になし（むしろ最新機能がないため敬遠されている）。

### Siki（siki）＋プラグイン関連
- **言及内容**:
  - Sikiのプラグイン「ai-prompt-marker v1.6」の使用（画像に[AIプロンプト有]表示、キャッシュ/ログ削除で動作改善）。
  - 画像ビューワ機能の自作（大量生成画像の高速確認＋即ゴミ箱ボタン、右スワイプで削除・左スワイプで保存など直感的操作）。
  - メタデータ取得や表示に関する不具合修正の議論。

- **ツールが選ばれている理由**:
  - 5ch閲覧中にAI生成画像のメタデータ（プロンプト）を素早く確認できる。
  - 生成画像の大量レビュー・整理作業が効率化できる（特にゴミ箱機能）。

### Kijaiノード関連
- **言及内容**:
  - KijaiのDepth ProcessorやConvRot関連ノードの使用例。
  - ComfyUI内でKijaiノードをDepth処理に置き換えて使用。

- **ツールが選ばれている理由**:
  - ComfyUIのネイティブサポート待ちをせず、先行して高機能ノードが利用できる。

## Web検索による参考情報
- **Siki**: 5ch（なんJ）専用のブラウザ/ビューア。プラグイン機構があり、`ai-prompt-marker`のような拡張がユーザーによって開発・共有されている。
- **Kijaiノード**: ComfyUIのカスタムノード作者として著名。量子化（INT8/ConvRot）やDepth処理、動画関連ノードを積極的に公開している。
- **ConvRot**: 整数量子化（特にW4A4/W8A8）を目的とした技術。ComfyUIコミュニティでINT8対応が進んでおり、速度向上を目的に採用されている。
