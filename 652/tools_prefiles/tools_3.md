## ツールに関する言及

### プロンプト・ワークフロー管理ツール
- **ComfyUI**:
    - 画像生成のワークフロー作成において主流のツールとして頻繁に登場。
    - LoRAの層別適用ノードや、スケジューリング構文によるプロンプト制御などが議論されている。
    - 画像生成時の高速化手法（Spectrum, Turbo, Easy Cacheなど）の実装や検証の場となっている。
- **NegPip**:
    - CFGスケール1などの「ネガティブプロンプトが効かない環境」で、ネガティブな要素を制御するために必須のツールとして推奨されている。
- **Kdenlive**:
    - 動画編集ソフト。モザイク処理などの作業に活用されている。

### 生成補助・自動化ツール
- **IrfanView**:
    - 生成画像の連番画像化のために使用。
- **LM Studio / チャットボット (Claude, ChatGPT, Gemini, Grok等)**:
    - プロンプトの生成、ワークフローの最適化、コード（LoRAの階層制御スクリプト等）の作成において、AIを「生成の助手」として活用する手法が極めて一般的になっている。
    - Gemini（特にモデルの知識に詳しい）やClaude（コーディング支援）が特に重宝されている。
- **自動モザイク用プラグイン/SAM (Segment Anything Model)**:
    - 動画や画像に対する自動モザイク処理技術として言及。ただし完全な自動化は難しく、現在も多くのユーザーが試行錯誤中。

## Web検索による参考情報

### 抽出ツール詳細補足
- **ComfyUI**: ノードベースのStable Diffusion専用Web UI。柔軟なノード構成により、複雑なプロンプトのスケジューリングや層別学習（LoRA）の制御が可能。
- **NegPip**: 主にComfyUIで使用される、ネガティブプロンプトの出力を特定のプロンプトノード経由で適用する手法。CFG1といった低CFG環境下での品質制御によく用いられる。
- **IrfanView**: 高速かつ軽量な画像表示・変換ソフト。大量の画像を連番画像として書き出す際に、画像生成コミュニティでも定番のツール。
- **Kdenlive**: オープンソースのノンリニア動画編集ソフト。Linux環境でも動作し、AI生成動画の微調整や編集に幅広く使われる。
- **SAM (Segment Anything Model)**: Metaが公開した、画像内の任意の物体を分離・特定できるAIモデル。ComfyUI等と連携し、特定領域へのモザイク処理やインペインティングのマスク作成に使用される。

### 生成AIトレンドの背景
- **「プロンプトエンジニアリング」から「AIによる指示出し」へ**: ユーザーが直接プロンプトをひねり出すのではなく、LLM（ClaudeやGemini）に画像の特徴を読み込ませ、最適なプロンプトを「考えさせる」手法が標準的になっている。
- **高速化の罠**: 「Turbo」等を冠する高速化LoRAは、生成時間を劇的に短縮する一方で、品質制御のためのプロンプトやネガティブプロンプトの効きが悪化するケースが多い。スレッド内では「こだわりの1枚を作るときはTurboを捨てろ」という教訓が共有されている。

## プロンプトテクニック・知見
- **スケジュール構文**: `[tag1:tag2:0.2]` のような記法を用い、生成ステップの途中でプロンプトを切り替える手法が、繊細な表現（髪の質感など）のために活用されている。
- **層別学習 (LBW: LoRA Block Weight)**: 特定のLoRAを適用する際に、モデル内のどの階層（ブロック）に強く効かせるかを調整する技術。特定の顔や画風を維持しつつ、服装やポーズの自由度を確保するために利用される。
