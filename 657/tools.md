# 元の本文
**生成AI関連ツール レポート**

### 1. ComfyUI（継続的変化・重点観測）
ComfyUIは複数回にわたり最も言及が多く、**前回・前々回から継続して環境管理・最適化・互換性**の観点で変化が観測されている。

**主な変化点**
- **dynamic vram**の実装により、動画生成速度が低下する事例（5秒動画で約1分遅延）が報告され、旧バージョンとの併用・切り替え運用が推奨されるようになった。
- バージョン更新（0.21.1→0.23.0など）でCLIP読み込みエラーやカスタムノードとの相性問題が顕在化。Kサンプラー停止やメモリ関連エラーもVRAM16GB環境で発生。
- **Kijai氏ツール群**（Ideogram4 Optimizations KJ、Patch Flash Attention KJ、uncondモデルnvfp4化など）の導入により、生成時間が大幅短縮（27.5秒→14.9秒、45%向上）し、VRAM使用量も削減された事例が新たに確認された。
- 環境管理面では、アップデートリスク回避のため物理ディスク（SATA→USB）で旧環境を保持する運用が新たに提案されている。

**選ばれている理由**
ノードベースの柔軟性とワークフロー再現性（PNGドラッグ&ドロップ）が評価されており、カスタムノードを多用する場合でも「自分で最適化できる」点が強み。一方でカスタムノード依存時は旧環境保持が現実的とされ、用途に応じた使い分けが求められている。

### 2. Ideogram（Ideogram 4.0 / Prompt Builder KJ）（新たに台頭）
前回まで散発的だった言及が今回大幅に増加し、**位置制御・JSONプロンプト**の観点で新たに注目されているツール。

**主な特徴・変化**
- JSON形式による位置指定（bounding box）が可能になり、自然言語では困難な複雑レイアウトを正確に制御できる点が複数回強調された。
- Kijai氏のPrompt Builder KJおよび軽量化ノード群との組み合わせにより、ComfyUIワークフロー内での実用性が向上（API版とローカル版の画質・速度差も確認）。
- テキスト生成精度や特定衣装・小物配置の安定性が高い一方、人物が交錯する複雑レイアウトでは属性ミックスが発生しやすい弱点も指摘されている。

**選ばれている理由**
「JSON学習モデルであるため位置制御が異常に強い」点が革命的と評価されており、手書き指示や複雑構図が必要な用途で特に選ばれている。

### 3. その他の変化・新登場ツール（簡潔）
- **Krita**：ComfyUI生成物との往復作業（inpaint）で新たに言及増加。手動マスクによる精密修正が評価され、位置制御ツールとしての補完役として機能。
- **Forge Neo**：シンプルな生成用途では高評価だが、複雑ワークフローではComfyUIに軍配が上がるという使い分けが明確化。
- **sd-scripts**：複数GPU対応によるLoRA学習時間短縮（3GPUで約半分）が新たに報告され、学習用途での実用性が向上。
- **LLM系（Gemini / ローカルLLM）**：ワークフロー自動生成やクラウドBAN回避目的で使用が増加。Grammarlyとの併用によるプロンプト品質向上も確認。

**総括**  
ComfyUIはバージョン更新と最適化ノードの進化により「速度・VRAM・互換性」のバランス調整が継続的な課題となっている。一方、Ideogram 4.0はJSON位置制御という新機能により、レイアウト重視の用途で急速に存在感を増している。KritaやKijaiツール群との組み合わせが、現在のワークフロー最適化の主流となりつつある。
