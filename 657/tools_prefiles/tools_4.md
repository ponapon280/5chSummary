**抽出されたツール関連話題（ComfyUI / Forge / 関連ノード・ワークフロー中心）**

- **ComfyUIの環境管理・容量対策**  
  アップデート時のリスク回避のため、古い環境をSSD/HDDに物理的に切り替えて保存したいという話。256GB SATAをUSB接続して環境ごとにディスクを差し替える運用が提案されている。

- **ComfyUI Manager / Portableのインストール**  
  python 3.13.12手動インストール版ComfyUIでRTX 3070環境において、旧Managerから問題なくインストールできた事例。Portable版は追加の手順が必要な可能性が指摘されている。

- **ComfyUIノードの最適化・アップスケール**  
  2xビデオアップスケールが大幅に高速化された事例。Seed VR2とRTXノードの比較が行われ、用途（静止画 vs 動画）によって使い分けられている。ComfyUI自体を「ノードを最適化していくゲーム」と位置づける意見あり。

- **Kijai氏ツール群（Ideogram4向け最適化）**  
  Ideogram4使用時に以下のツール/ノードを組み合わせることで大幅高速化（27.5秒→14.9秒）：
  - uncondモデルをnvfp4に変更
  - flash-attnインストール
  - Patch Flash Attention KJ
  - Ideogram4 Optimizations KJ  
  オフロードが不要になり45%のスピードアップが確認された点が評価されている。

- **ComfyUIワークフロー作成・管理**  
  - ノードを自分で追加して効率化するメリットが複数指摘。
  - 他人のワークフローを理解するコストの高さ（カスタムノードの多さ、設計思想の把握）。
  - AIにワークフローを生成させることへの賛否（本末転倒という意見と、json扱いの苦手さを補うという意見が対立）。
  - サブグラフの操作方法（階層表示をクリック or Escキー）。

- **Forge（Forge Neo）**  
  Forge Neoを高評価する声あり。一方で、画像生成以外の作業（LoRAマージなど）ではComfyUIの方が柔軟という指摘。

- **EasyWAN / Wan関連ワークフロー**  
  Wan動画生成のためにComfyUI必須という認識。EasyWAN22は「動かすだけなら簡単だが、理解して改造しようとすると複雑（InfernoWAN化）」という評価。モザイク処理の移植事例あり。

- **ポイントモザイク系ワークフロー**  
  EasyWANのモザイク処理をComfyUIワークフローとして移植・利用する事例。検出器依存の限界を認識しつつ、AE自作に切り替えた例も。

- **その他のツール・運用**
  - Docker使用時の環境リセット対策として高速クローン環境を構築。
  - VSCode + JSON/YAML/toon形式でのワークフロー管理（コメントの扱いやすさ、トークン削減）。
  - Tadaupでのメタデータ残存の不安定さ（imgbb.com推奨の声）。

**特にツールが選ばれている理由として抽出された点**
- ComfyUI：ノード単位での細かい最適化・カスタムが可能で、動画・アップスケール・特殊処理に強い。
- Kijaiツール群：特定のモデル（Ideogram4）でオフロード削減と大幅速度向上を実現。
- EasyWAN系：初心者向けの簡易さはあるが、理解・改造のしにくさがネック。
- Forge Neo：シンプルな用途では高評価だが、複雑なワークフローではComfyUIに軍配。

（モデル名・Anima・WAI・Illustriousなどの話題はすべて除外済み）
