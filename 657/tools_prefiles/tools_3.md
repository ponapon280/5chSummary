**ツール関連の抽出結果**

**ComfyUI（comfy）**
- ワークフローをJSON/ノード形式で管理・呼び出すことで全体がスッキリし、満足度が高い。
- PNGドラッグ&ドロップでワークフローが再現される機能が便利（ただしカスタムノードの互換性に注意）。
- 位置指示や複雑なレイアウト制御に強いノード群を組み合わせやすい。
- TensorRTやNVIDIA RTXノードとの連携で高速化（特にフレーム補間・アップスケール）。
- バージョン差異が大きく、0.21.1→0.23.0などでCLIP読み込みエラーやdynamic vram周りの互換性問題が発生しやすい。
- 理由として「ノードベースの柔軟性」「ワークフローの再現性」「LLM連携ノードの拡張性」が挙げられている。

**NanoBanana**
- 抜歯済歯茎画像の生成やパペット変形後の画像生成に使用。
- Kritaとの往復作業（inpaintガチャ）の中で中間生成ツールとして機能。
- 理由として「特定の部位（歯茎など）の生成精度が高い」点が言及されている。

**Ideogram（Ideogram 4 / Ideogram4）**
- JSON形式での位置指定（例: school deskを2つ離して配置）が非常に便利で、複雑なレイアウトが自然言語より正確に制御できる。
- シンプルな記述でも詳細を自動補完（小さい文字やロゴの追加）してくれる。
- Prompt Builder（Kijai氏のIdeogram 4 Prompt Builder KJ）との組み合わせでワークフローが効率化。
- スク水などの特定衣装も「どんな水着か」を入れると安定して出力。
- 理由として「JSON学習モデルであるため位置制御が異常に強い」「自然言語では通りにくい配置が簡単にできる」点が複数回強調されている。
- ただし人物が交錯するような複雑レイアウトでは属性ミックスが発生しやすいという弱点も指摘。

**Krita / kritaAI**
- inpaint領域を手動マスクして精密に修正する用途で使用。
- 3Dモデル合成やパペット変形後の画像の馴染ませに活用。
- 理由として「手動マスクによる細かい制御が可能」「ComfyUI生成物との往復作業がしやすい」点。

**Photoshop / Firefly**
- 位置指示インターフェースが最初からある（外部ツール使用時も同様）。
- Fireflyは将来的に位置指示機能が強化されると予想されている。
- 理由として「位置制御の完成度が高い既存ツール」として言及。

**Forge Neo**
- WAI-Animaサンプル生成に使われており、ComfyUIより手軽に扱える場合がある。
- 理由として「デフォルト設定で高品質が出やすい」点。

**その他のツール関連言及**
- ai-toolkit：LoRA作成用。
- ComfyUI Manager / カスタムノード：TensorRTやRIFE VFIの導入で大幅高速化。
- 全体の傾向として「位置制御・ワークフロー再現性・高速化ノード」の3点でツールが評価されている。

**モデル関連は一切抽出していません**（Anima、WAI、illustrious、Qwen-Imageなどの言及は除外）。
