**抽出されたツール関連話題（モデル名・モデル性能の話題は除外）**

- **ComfyUI（comfy）**  
  - ワークフロー内で完結させたいという理由で選択（「ワークフローですべてを完結させたい」「ポン出しに命をかける」）。  
  - 公式ワークフロー使用＋ultrasharpでのアップスケール運用。  
  - 実行時間表示ノードの要望（Queue履歴で代用可能と判明）。  
  - カスタムノード作成の容易さ（グループ位置入れ替えノード、生成中サムネ表示ノードなど）。  
  - Managerの更新状況やInspirePackの対応待ちの話題。  
  - ideogram使用時のKijai軽量化セット・flash attention・Ideogram4 Optimizations KJノードの組み合わせでVRAM削減・高速化。  
  - Dual Model CFG Guiderから通常CFG Guiderへの切り替えによるVRAM大幅削減（55%程度）。  
  - TensorRT系アップスケールノード（Upscale Tensorrt）の解像度上限問題。

- **sd-scripts**  
  - 複数GPU対応でLoRA学習時間を短縮（3GPUで約半分）。  
  - ANIMA学習時のflow_shift設定に関する実践報告。

- **EasyAnima**  
  - Anima関連の学習・運用で全乗っかりで使用。更新が早い点を評価。

- **OrthoLoRA / T-LoRA（階層学習ツール）**  
  - sd-scripts系とは別の専用学習ツールとして紹介され、組み合わせ推奨の声あり。  
  - 手が崩れにくい学習を期待しての言及。

- **nano-banana（ナノバナナ）**  
  - ローカル環境で同等品質が出せる点を評価（「ローカルでナノバナナ級は普通に嬉しい」）。

- **Firefly**  
  - 生成速度が遅いためComfyUIワークフロー組むまでの手間を嫌う理由で敬遠。

- **ideogram（ComfyUI統合版）**  
  - ComfyUIテンプレートWFでのCFG調整・Quality設定の試行。  
  - Kijai軽量化セット使用でガビガビ画質改善・VRAM削減を推奨。

- **カスタムノード全般（ComfyUI）**  
  - 既存ノードで実現できない機能をAIに作らせる運用が活発。  
  - 3桁未満のマイナーカスタムノードはウイルスリスクが高いという警戒論。
