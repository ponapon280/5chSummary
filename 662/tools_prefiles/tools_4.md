**抽出されたツール関連話題（ComfyUI/webUI系中心）**

**ComfyUIのtempフォルダ動作と保存先変更**  
- comfy起動時にtempフォルダへ画像/動画がPNGで大量保存される挙動（画像プレビュー、Resize Image v2通過時など）。  
- SSD寿命を気にしてHDDへジャンクション作成した事例。  
- 起動オプション`--temp-directory D:\tmp\`でtempフォルダの場所を任意指定可能（古くからこの仕様）。  
- queue一覧のView表示時にもtempが使用される。  
- 理由：不要な書き込みを減らしたい、SSD保護。

**save-image-extended-comfyuiノード**  
- outputフォルダへwebp形式で永続保存したい場合に推奨。  
- ノードのデフォルトがPNGのため、設定変更でwebp出力に切り替え可能。  
- tmpフォルダにwebpを直接保存するのは公式非対応。  
- 理由：PNGよりファイルサイズを抑えつつoutputへ保存したい要件に対応。

**FaceDetailerの設定**  
- ComfyUIでanima使用時にFaceDetailerの設定をどうするかの話題。  
- 理由：生成品質向上のための微調整目的。

**BiRefNet + sam3を組み合わせた背景透過ワークフロー**  
- 人物と背景を分離し、2フレームごとに保存する動画透過処理。  
- Professional Background Removerを参考に自作。  
- 理由：髪の交差処理など、既存ノードのエラーを回避しつつ用途に合わせたワークフローを構築。

**kjnodes（Ideogram 4 Prompt Builder KJ）**  
- Krea-2でもjsonプロンプトが扱えるため流用可能（v2以降は使い方が変わった）。  
- 理由：jsonプロンプト対応によるプロンプト記述の容易さ。

**その他のツール言及**  
- webUI (A1111)：tempフォルダの挙動比較（ComfyUIとの違い）。  
- Qwen3-VL-4B：Krea-2のテキストエンコーダーとして使用（画像生成以外の文脈で言及）。検閲解除版への差し替え可能性を議論。  
  → Qwenシリーズの非画像生成用途（テキストエンコーダー）として抽出。

**ツールが選ばれる主な理由のまとめ**  
- 保存形式・保存先の柔軟性（webp出力、tempフォルダ移動）によるSSD保護・利便性。  
- ワークフロー構築の自由度（ノード組み合わせで背景分離・動画処理など）。  
- jsonプロンプト対応による記述のしやすさ。  
- 起動オプションによる環境カスタマイズ性。

モデル名・バージョン・新サービス関連は一切抽出していません。
