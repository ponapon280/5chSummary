**ツール関連の抽出（モデル名・バージョンは除外）**

### ComfyUI（comfy）
- **話題のポイント**:
  - civitaiの画像に埋め込まれたワークフロー（WF）を直接読み込む方法（右下の「COMFY: xx Nodes □」マークをクリック→Pasteで即読み込み）。
  - カスタムノードを活用したプロンプト管理・トリガーワード管理。
  - Google翻訳ノードの使用例（自然言語→部族語の変換補助）。
  - InpaintModelConditioningノードを使ったマスク限定inpaintの提案。
  - anytest + inpaintのControlNet併用ワークフロー（狙った箇所のみ編集）。
  - 複数インストールを避けつつノードを使い分ける方法の模索。

- **選ばれている理由**:
  - ワークフローの埋め込み・コピペが非常に楽で、再現性が高い。
  - カスタムノードで柔軟に拡張できる（プロンプト管理など）。
  - 画像から直接WFを読み込めるため、テキストに書かれていない設定（sampler/schedulerなど）も確認可能。

### civitai
- **話題のポイント**:
  - ワークフロー埋め込み画像の探し方（WFコピペボタンが出ている画像を優先）。
  - プロンプト詐欺・クオリティ詐欺の多さ（実際のプロンプトと違う、アップスケーラー使用を隠すなど）。
  - 参考にする際の注意点（metadataに本物のWFが入っているかを確認）。

- **選ばれている理由**:
  - WFが画像に埋め込まれている場合があるため、視覚的に確認しやすい。
  - ただし詐欺が多いため、盲信せず画像を落として検証する使い方が推奨されている。

### LM Studio
- **話題のポイント**:
  - 自然言語プロンプト作成時の補助ツールとして言及。

- **選ばれている理由**:
  - ローカルで手軽にLLMを使えるため、プロンプト作成の参考に適している。

### その他のツール・サービス
- **Danbooru**:
  - タグ辞書の参照（サイト内の「?」マークで辞書引き）。
  - 部族語（danbooruタグ）ベースのプロンプト作成時の補助。

- **Gemini（無料版）**:
  - ワイルドカード大量生成、プロンプトの有効性テスト、画質・画風・ポーズなどの生成。
  - テンプレートを用意して一括生成させる使い方。

- **HuggingFace**:
  - ファイルの代替取得先として言及（特定のファイルが消えた場合の確認場所）。

- **aibooru**:
  - civitaiの代替候補として提案（プロンプト参考としての優位性）。

- **easyreforge / forge neo**:
  - Inpaintの挙動が安定しているため、差分生成やマスク限定編集で使用。

- **Anima-Lora-Factory / Lora_GUI**:
  - LoRA作成時の学習設定・タグ処理の参考ツールとして言及。

- **Zuntan提唱のanytest + inpaint手法**:
  - ComfyUI上でのキャラクター差分作成に活用。

**抽出されなかったもの**:
- Anima、リアス、illustrious、FLUX、Z-Image、LTXなどモデル関連の話題はすべて除外。
- Qwenシリーズに関する非画像生成話題も該当なし。
