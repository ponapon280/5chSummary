# 元の本文
**生成AI関連ツール レポート**

### 1. ComfyUI（最も変化・注目が集中しているツール）
ComfyUIは前回・前々回から引き続き「WebUI/Forgeからの移行」が進行中であり、今回も移行の具体的な理由と運用上の変化が複数報告されています。

**主な変化点**
- **バージョン更新の必要性**: Krea2などの新モデル利用時に「v0.25以前ではClipタイプ設定ができずエラーになる」ため、v0.26.0以降への更新が必須となった事例が確認されました。これにより既存カスタムノード（ねぎぴっぴなど）が使えなくなるケースも発生しています。
- **tempフォルダの挙動と対策**: 起動時にtempフォルダへPNGが大量保存される問題に対し、`--temp-directory`オプションやHDDへのジャンクション作成でSSD保護を図る事例が増えています。
- **保存形式の拡張**: `save-image-extended-comfyui`ノードの採用により、outputフォルダへのWebP出力が可能になり、ファイルサイズ削減と永続保存の両立が進んでいます。

**選ばれている主な理由**
- ノード単位での柔軟なワークフロー構築が可能（BiRefNet+sam3による背景透過動画処理、QwenImageEditWFの流用、ConditioningKrea2Rebalanceによる微調整など）。
- civitai画像に埋め込まれたワークフローをワンクリックで読み込める利便性。
- 新モデル対応や保存・運用面でのカスタマイズ性が高い点が評価されています。

### 2. Animerge（新機能追加により再注目）
前回まであまり詳細が触れられていなかったAnimergeについて、複数回の更新と新機能の追加が報告されました。

**主な変化点**
- GUIの見た目調整、翻訳UI追加、マージタブのUI変更（プルダウン→エントリー方式）、実行ログ表示、ADDifTプリセット追加。
- **DPOモードの新規追加**: 良い画像・悪い画像のペアでモデルを好みに寄せる簡易調整機能。LECOとは異なり「関連ワードへの経路遮断」が特徴で、学習時間も1分程度と短い。

**選ばれている主な理由**
- 「カユいところに手が届く」特殊効果調整が手軽にできる点が特に評価されており、カスタムノードとして「金取れるレベル」との声もあります。

### 3. その他の変化・新登場ツール（簡潔に）
- **nano-banana**: LoRA学習時のデータ水増し用途で言及。SDXL時代から続く特有の強調傾向を逆手に取った素材厳選の工夫が報告されています。
- **LM Studio / Gemini（無料版）**: プロンプト作成補助ツールとして安定して使われており、特にGeminiはワイルドカード大量生成や有効性テストに活用されています。
- **kjnodes（Ideogram 4 Prompt Builder KJ）**: Krea-2のjsonプロンプト対応により流用が進んでいます。

**まとめ**  
前回からの主な変化は「ComfyUIの運用面（temp/保存形式/バージョン管理）」と「AnimergeのDPOモード追加」に集約されます。ComfyUIは移行完了後の細かな運用改善が、Animergeは新機能による利便性向上が、それぞれ選ばれる理由となっています。
