## 生成AI関連ツール抽出結果

### 画像生成・ワークフローツール
#### ComfyUI (comfy)
- AIがどのように生成されているかの理解があれば簡単なコンテンツであると言及されている。
- 起動直後の画面が「黒い方眼紙」のような見た目であると表現されている。
- ワークフロー（WF）ファイルをドラッグ＆ドロップすることで設定を読み込める点が挙げられている。
- デフォルトでは生成の都度手動で保存する必要があり、A1111（webUI）のような自動保存に慣れているユーザーには不便に感じられることがある。

#### WebUI / A1111 / Forge
- A1111は生成画像がフォルダに自動的に溜まっていく仕様であると言及されている。
- Forge Neoの更新により`generation_parameters_copypaste`が削除され、一部の拡張機能（Dynamic Promptsなど）が動作しなくなったという報告がある。
- Forge Couple（SD Forge Attention Couple）を使用することで、Animaにおいて領域書き分けによる複数キャラの配置が便利になるとされている。

### 学習・キャプション・タグ管理ツール
#### Kohya_lora_param_gui
- LoRA作成に使用されており、キャラや画風の作成に活用されている。

#### Anima-LoRA-Factory
- AnimaのLoRA作成ツールとして利用されており、画風LoRAの作成に成功した例がある。

#### TagGUI
- キャプション付けを非常に楽に行えるツールとして推奨されている。

#### anima-tagger caption
- キャプション作成に使用されたが、Windows/Ubuntu環境でテンソル作成時の次元エラーが発生したとの報告がある。

#### その他自作・特定ツール
- データセット準備用の自作ツール（バイブコーディング製）が提示されている。
- Nova (Anima版) というツールに関する言及がある。

### コーディング・エージェントAIツール
#### Codex
- PC内部で直接作業し、リソースを消費しながら逐一ユーザーに許可を求める仕様であるため、不便で面倒であると感じられている。
- プログラミングやブラウザ操作の性能が不十分であるとの評価がある。

#### Cursor / Claude Code / AG
- Codexと比較して、直感的に使用できるツールとして挙げられている。

### その他AIサービス・プラットフォーム
#### Civitai / Civitai Comics
- モデルのダウンロードや、生成した作品の公開（comic機能）に使用されている。

#### LLM (Claude, ChatGPT, Gemini, Grok)
- キャプション作成や自然言語によるプロンプト案の生成、コードの修正（zip渡し）などに利用されている。
- 規制の緩さや文章のクオリティについて比較されており、特にAPI経由での利用や脱獄手法について議論されている。
