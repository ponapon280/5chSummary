# 元の本文
**生成AIモデルに関するレポート**

### 人気モデルの概要
本テキストから、**Anima（特にAnimaBase + Turbo LoRA）** が圧倒的に最多言及されており、現在の主流モデルであることが明確に読み取れる。漫画・アニメ調画像生成、特にLoRAを多用するワークフローで支配的な地位を占めている。

### 状況の変化と重点モデル

**Animaの台頭（最大の変化）**  
Animaは複数ログで「メインエンジン」「SDXLの後釜」と位置づけられ、Illustrious（リアス）からの明確な移行が進んでいる。選ばれている主な理由は以下の通り：

- 自然言語プロンプトへの追従性が高く、複雑な構図や強調タグでも破綻しにくい
- LoRA（特にキャラLoRA・画風LoRA）との相性が非常に良く、学習効率・再現性が高い
- 漫画制作での実用性（50ページ規模を少ない生成枚数で完結できる生産性）
- デフォルメ（低頭身）＋large breastsのような従来難しかった表現の打率向上

Illustrious時代には苦手だった領域で改善が見られたため、乗り換えが進んだ形だ。

**Illustrious / リアスからの移行**  
Anima登場以前の定番だったIllustriousは、現在は比較対象として言及されることが多くなっている。原作画風再現性やv-predの安定性では一定の評価を残しているが、強調耐性やデフォルメ表現でAnimaに劣るとの声が目立つ。派生モデル（AnyIllustrious-XL）もLoRA学習用途で残存している程度である。

**Pony系（ebara）の位置づけ変化**  
ebaraシリーズは「pony時代を象徴するモデル」として過去形で語られることが増えている。表現力の高さは評価されているが、Anima登場により相対的に話題が減少し、「ebaraありきだったのに…」という感慨が散見される。

**その他の言及（変化が小さいor補助的）**
- **NovelAI（NAI）**: inpaint性能などで一定の評価は残るが、全体的な品質・一貫性でローカル勢（特にAnima）に水をあけられているとの認識が強い。
- **Qwen**: 主にVLM（キャプショニング）やText Encoderとしての用途で登場。画像生成モデルとしての言及はほとんどない。
- **NVIDIA Cosmos系**: Animaのベース技術として間接的に言及され、今後の進化への期待材料として登場。

### まとめ
前回・前々回と比べて最大の変化は **Illustrious → Anima** への明確な移行であり、Animaが「プロンプト理解力・LoRA親和性・実用性」の三点で支持を集めている。Ponyも過去の人気モデルとして言及されるものの、勢いはAnimaに取って代わられている状況である。
