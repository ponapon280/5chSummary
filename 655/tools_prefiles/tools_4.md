**抽出されたツール関連話題（ComfyUI / webUI系 / Krita連携など）**

- **ComfyUIのバージョンアップと挙動変化**  
  v0.22.0 → v0.23.0で「高shift・低denoise時のi2i変化が弱くなった（正常化）」「denoise調整が効きやすくなった」という報告。Python 3.10→3.13に上げたところ、TensorRTアップスケールの処理時間が30秒→2秒になった事例あり。ComfyUI公式のMultiGPU対応（MultiGPU CFG Splitノード）で並列加速が期待できる点が注目されている。

- **ComfyUIのワークフロー・ノード活用**  
  - rgthreeのcontextノードや多コネクタノード、crytoolsのpipeノードを多用した整理されたワークフローが複数言及。  
  - Anima LoRA filter (Blocks)ノードを2個使ってキャラLoRAと絵柄LoRAを層別に適用する試み。直列接続時の効きが弱まる問題や、ブロックカットして新LoRAとして保存→マージする手法が議論された。  
  - 生成中プレビュー窓のみを表示するカスタムノードの要望。

- **Krita + ComfyUI連携**  
  Krita上でComfyUIと同じ絵柄・AnyTestでの追従性を再現できた報告が複数。Kritaのプラグイン経由でComfyUIワークフローを呼び出し、結果を新規レイヤーとして返す形が理想とされ、webUIよりComfyUIとの相性が良いという声が多い。カスタムサーバー選択時の挙動や、必要なモデルだけ残す運用についても触れられている。

- **Forge / webUI系**  
  ForgeNeoでAnimaが動くという情報。webUIからKrita連携を試した人の感想として「i2iやアップスケールはやりやすいが、ComfyUIほどの柔軟性はない」という比較がなされた。

**特に「このツールを選ぶ理由」として抽出された点**
- ComfyUI：ノードベースで細かい制御（LoRAの層別適用、ブロックウェイト、MultiGPU並列処理）が可能で、速度向上やワークフローの再現性が高いため。Pythonバージョン変更による実測速度改善も評価。
- Krita+ComfyUI：ドローソフトから直接AnyTest向け素材を渡しやすく、生成後の編集まで一貫して扱える点。webUI単体より「描き込み」や「追従性」で有利。
- ForgeNeo：Animaを動かすための実用的な選択肢として言及。
