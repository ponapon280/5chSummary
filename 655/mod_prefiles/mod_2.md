**抽出結果**

### anima（Anima / AnimaBase / Turbo LoRA）
ログ内で圧倒的に多く言及されているモデル。漫画制作のメインエンジンとして使われている。

**主な話題・評価**
- 7割程度をanimaのポン出し→ControlNet（主にLineArtやanytest）で高画質化して使用。
- プロンプト理解力が高く、竿役（男性キャラ）が入る複雑な構図でも比較的破綻しにくい。
- 「anime screenshot」タグを強く入れることで、深夜アニメらしい塗りに寄せられる。
- Turbo LoRAの使用については意見が分かれていて、「絵柄安定に効果がある」「多様性が減る」という両方の声あり。
- 学習効率が非常に高いため、データセットに不要な画像を入れると影響が大きく出やすい（特に画風LoRA作成時）。

**選ばれている理由（ログから読み取れる点）**
- プロンプトだけでかなり意図した構図・ポーズが出やすい。
- 塗りや影の処理が比較的自然（ただし厚塗り絵師の作風を再現しようとするとグラデーションが邪魔になる場合あり）。
- 50ページクラスの漫画を1200枚程度の生成で作り上げられる生産性の高さ。

このログから、除外モデル（anima系含む）を除いた「モデル」関連の話題は以下の通りです。

### 1. SDXL（Stable Diffusion XL）
- 話題：画風LoRA学習時の背景処理について
- 内容：白背景での学習がSDXLとanimaのどちらでも有効か質問されている。

### 2. gemma4（Gemma系ローカルLLM）
- 話題：4chanビューワーのローカルLLM化における猛虎弁翻訳
- 内容：猛虎弁再現性能は非常に高いが、処理速度が遅い（300レスで12〜20分程度）という問題が複数指摘されている。
- 選ばれている理由：猛虎弁への適応力（ネイティブレベルで再現可能）が評価されている。

### 3. GPT-4.1-mini（GPT-4o-mini相当）
- 話題：ローカルLLMとの性能比較
- 内容：200レス程度の翻訳を1分程度で処理でき、猛虎弁も比較的安定して出力できると評価。
- 選ばれている理由：速度と実用性のバランスが良いため、ローカルモデルより優先されるケースあり。

### 4. DeepSeek API
- 話題：Niki-4chanビューワーでの利用
- 内容：API経由で猛虎弁翻訳ツールを使用した報告あり。
- 選ばれている理由：記載なし（単に動作確認として使われた）。

### 5. Irodori-TTS Voice-Design v3
- 話題：新バージョンのリリース
- 内容：v3で参照音声＋テキスト＋キャプションテキストの併用が可能になったと指摘。
