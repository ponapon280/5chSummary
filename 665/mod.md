# 元の本文
**生成AIモデル動向レポート**

### 流行モデルのまとめ
複数のログから、**Krea2**が現時点で最も急上昇しているモデルとして突出している。リアル寄りの高精細表現とINT8量子化による高速化が評価され、コミュニティの関心が集中している。一方、アニメ・二次元領域では**anima**が依然として主力であり、Krea2との住み分けが明確化しつつある。動画生成では**Wan2.2**と**LTX**が二大勢力として定着し、特にWanはキャラ一貫性で支持を集めている。NovelAI（特にv5）はクラウド勢としてエロ漫画生成への期待が高まっている。

### 状況が変化した・新たに登場したモデル

**Krea2（Krea2 Turbo / raw / INT8版含む）**  
前回までのログと比べて言及数が最も増加しており、新星として急速に台頭している。主な変化点は以下の通り：
- INT8量子化やConvRot版の最適化が進み、RTX 30xxシリーズでも実用的な速度（1MPで5.5〜7.4秒程度）が出せるようになった点が強く評価されている。
- リアル系・布の質感・日本人描写で優位性があり、特にビフォーアフター（2分割生成）での一貫性が高い。
- Forge Neo対応やmusubi-tuner連携によるLoRA学習環境の整備が進み、「本家モデルの完成度が高い」との声が多い。
- 一方でNSFWの扱いや派生モデルのキャラ再現性低下が指摘されており、「本家＋脱獄/LoRA」の組み合わせが主流になりつつある。

**anima（AnimaBase / animaDex含む）**  
Krea2登場による影響が顕著に見られるモデル。状況の変化として：
- Krea2が実写・リアル寄りで台頭したことで、animaは「二次元・エロ特化」としてのポジションがより明確化。
- LoRA蓄積量の多さと学習の安定性（dim64/alpha32など）が引き続き強みとして挙げられ、Forge Neo + EasyForgeNeoの組み合わせで「人権を得た」との評価。
- Krea2との比較で「服装の崩れにくさ」や「爆乳時の乳首露出制御」が優位点として言及されるケースが増えている。

**Wan2.2（WAN22）**  
動画生成領域でLTXとの二強構造が定着し、状況が変化している：
- 顔・体型の一貫性が高く「ぐりぐり動かしても別人化しにくい」点がLTXとの明確な差別化要因として強調されるようになった。
- SmoothMix + DaSiWaなどのワークフローや3段KSamplerが流行し、i2v性能の実用化が進んでいる。
- VRAM管理（Block Swap、Dynamic VRAM）の話題も活発で、64GBメモリ環境の重要性が指摘されている。

**NovelAI（NAI v5）**  
クラウドサービスとして新たに注目度が上がっている：
- 自然言語プロンプトの理解度向上とエロ漫画生成（t2t・コマ割り・擬音）への期待が複数ログで語られている。
- 「ローカルより明らかに効率が高い」という実用性評価が増加しており、v5でのさらなる進化に注目が集まっている。

### 補足
illustriousは主にanimaへの画風移植用学習ソースとしての位置づけが強く、直接生成モデルとしての言及は減少傾向にある。LTXはWanとの比較で「激しい動きは強いが一貫性で劣る」という評価が定着しつつある。全体として、Krea2の登場により「リアル vs アニメ」「クラウド vs ローカル」の住み分けが進行している状況が読み取れる。
