**抽出されたモデル関連話題**
### 1. **Illustrious（リアス / ill / IL）**
- 本家モデルと派生FTモデルの違いが複数回指摘されている。
- **選ばれている理由**:
 - FTモデルは「エロに特化しすぎてて、品質タグだけで生成しても半分くらいおっぱいが出てる」ため、**本家モデルを好む**声あり。
 - int8量子化の対象として具体的に挙げられており（6.3GB → int8変換）、速度向上を目的とした実験が行われている。
- int8変換後の扱い（UNET単体 vs 単一モデル化）についても言及あり。
### 2. **Anima（anima baseV10）**
- 爆乳生成時の**乳首の露出制御**で評価が高い。
- **選ばれている理由**:
 - 「明示しなければ爆乳設定でも乳首が出ないのがむしろ普通」「no nipplesで解決しやすい」。
 - Krea2などのリアル系モデルと比較して「服装が崩れにくい」という評価。
- mxfp8変換による速度向上の可能性についても触れられている。
### 3. **Krea / Krea2**
- ログ内で最も言及が多いモデル。
- **選ばれている理由**:
 - リアル系画像生成、特に**ビフォーアフター（2分割）**での一貫性が高い。
 - 爆乳表現の扱いがシビア（huge breasts以上で乳首が出やすい）という欠点も指摘されている一方で、「実写1枚絵を2分割で綺麗に繋げられる」と実用的な使い方が紹介されている。
- LoRAとの組み合わせや、gpt-image2との併用（ZITで出せない構図をgpt-image2で生成→Krea2でi2i）も話題に。
### 4. **Wan / WAN2.2**
- 動画生成モデルとして登場。
- **選ばれている理由・使い方**:
 - High/Lowの2系統があり、LoRAの適用方法（Highのみ / Lowのみ / 両方）が議論されている。
 - 最終的に流行ったワークフローとして「Highをlightx2v無し/有り、Lowをlightx2v有り」の**3段KSampler**が挙げられている。
### 5. **Z-Image Turbo（ZIT / ZIE）**
- Krea2やgpt-image2との併用で言及。
- **選ばれている理由**:
 - 単体では出せない構図をgpt-image2で生成し、ZITでi2iする使い方が紹介されている。
- Turbo版という特性から、速度を重視したワークフローで使われている模様。
### 6. **NovelAI（NAI）**
- クラウドサービスとしての言及が中心。
- NAIv5でエロ漫画生成が可能になるかどうかという将来性について触れられている。
- ローカル勢からは「クラウドサービス全般を敵視する」意見も見られる。
### その他
- **QIE**: 1回だけ登場（「QIEでも服消し飛ぶし」）。Qwen-Imageの略かと思われるが、詳細な話題はなし。
- **FLUX / LTX**: 該当する言及なし。
- **int8量子化**: ほぼ全てのモデルで横断的に話題になっており、特に**速度重視**で選ばれる傾向が強い（RTX20/30シリーズ救済としての側面も）。

**抽出された「モデル」関連話題**
### 1. Krea / Krea2
- リアル系画像生成で頻出。爆乳表現時の乳首露出問題（huge breasts指定でほぼ確実に出る）が複数投稿で議論され、「large breastsまでなら問題ない」「no nipplesでも効かない」「Krea2でも乳首が出ないケースもある」との報告あり。
- 利点として「実写1枚絵の2分割ビフォーアフターで背景・ポーズの一貫性が高い」「LoRAを混ぜてGPT-image2風の質感に近づけられる」と評価。
- 選択理由：リアル寄りの出力が安定しており、Krea2登場以降にリアル系需要が増えた文脈で言及。二次元モデルとは異なり、学習元の実写影響で露出制御が難しい点が特徴として指摘されている。
### 2. 量子化（int8 / fp8 / mxfp8 / nvfp4）
- int8変換による速度向上とVRAM削減が活発に議論。Illustrious（除外対象）の例が挙げられたが、概念として「3000番台=int8、4000番台=fp8、5000番台=mxfp8/nvfp4」が各世代最速の選択肢とされる。
- RTX20/30シリーズではint8が救済策として機能し、ComfyUIのネイティブ対応で速度が向上。古いint8は精度・速度で偽物扱いだったが、最近のものは「本物で高速」との認識。
- 選択理由：特にRTX30シリーズ以前のユーザー向けに「グラボ新調を逃した救済」としてint8が見直されており、fp8非対応GPUでの実用性が高い。
### 3. SD / SDXL + Turbo系最適化
- CFG値と生成時間の関係、ネガティブプロンプトの仕組み（attention 2回走らせて差分を取る）、TurboのCFG=1化による高速化が説明されている。
- 選択理由：Turboは「ステップ蒸留＋CFG1化」の組み合わせで速度を稼ぐため、ネガティブプロンプトが使えない代わりに実用速度を優先する場面で言及。
**除外対象モデル**（NovelAI、illustrious、FLUX、Wan、Qwen-Image、anima、Z-Image、LTX）は話題から除外して集計済み。
