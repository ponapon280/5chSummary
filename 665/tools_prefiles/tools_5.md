**抽出されたツール関連話題（モデル除外）**

- **ComfyUI（Comfy）関連**
  - サブグラフ（Subgraph）の安定性・活用について複数言及。以前は編集不能になるエラーが発生してトラウマになっているユーザーもいる一方、最近は安定してきたという声あり。
  - サブグラフのメリット・デメリット：LoRA変更時の手間、処理層のまとめ（プロンプト正規表現やファイル名決定処理など）に有効。大量ノードを隠した後のsetノード使用は避けるべきという意見。
  - サブグラフをモジュール化して使用しており、「サブグラフなしのワークフローには戻れない」という声多数。モジュール化は正しい使い方と評価されている。
  - INT8対応：ComfyUIがネイティブ対応したことで速度向上。Comfy-kitchenの更新が必要で、更新によりRTX 20シリーズでも高速化可能という情報。

- **Krea / Krea2関連**
  - 実写画像の2分割ビフォーアフター生成や、人物LoRA使用時の背景・ポーズ一貫性で評価。
  - ライティング変更（Kleinとの組み合わせ）やLoRA混合で「gpt-image2っぽく」できる点が好評。
  - エロ寄り学習がされているため出来ることが増える点がメリットとして挙げられている。
  - ダウンロードが19%で止まるなどの技術的トラブル報告あり。

- **その他のツール・機能**
  - Batch Count / Generate forever（連続生成機能）の挙動説明。拡張機能はREADME確認を推奨。
  - CFG値変更による生成時間への影響（KreaではCFG上げで時間が伸びるが、Turbo系は影響小）。
  - tadaup（画像アップロードサービス）のアクセス拒否エラー（XServer関連）について複数報告。

**特にツールが選ばれている理由として抽出された点**
- ComfyUI：INT8による速度向上、サブグラフによるワークフローのモジュール化・再利用性。
- Krea2：実写寄りの出力のしやすさ、LoRA混合時の柔軟性、ライティング調整などの編集機能。
