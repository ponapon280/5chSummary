**抽出結果**

### ツール関連の話題（モデル名は除外）

- **Forge Neo / EasyForgeNeo**
  - 更新後にVAEとtext encoderがmodelsフォルダーに日付付きで生成され、既存のVAE/text encoderをコピーすることでAnima生成が復旧した事例。
  - 更新（5月5日および5月末）でDynamic PromptsとWD14 Taggerが動作しなくなる問題が発生。作者側がui.pyの旧モジュール参照変更とinterrogator.pyのuse_cpu互換化で対応済み。
  - 理由として「Forge Neoで拡張機能の互換性を保ちつつ最新機能を使いたい」点が挙げられている。

- **ComfyUI / Comfy**
  - v0.27.0でint8モデルサポートが追加され、ModelMergeKrea2ノードが新たに追加された。
  - ModelSamplingAuraFlowノノードを入れずにAnimaを使っていた事例（Shift値調整ができなかったが生成自体は可能だった）。
  - WAN2.2ワークフロー作成時にFast Groups BypasserでStart/End、LightX2V、NSFW LoRA、Sage Attention、フレーム補間、高解像度化などの機能を切り替え可能にしている。
  - 理由として「ノードベースで細かくワークフローを制御したい」「長尺動画生成やi2v（鏡音リン疑似フェラなど）の高度な制御を行いたい」点が挙げられている。

- **nano-banana（ナノバナナ）**
  - NAIv5が「仕組みがナノバナナチックになる」との言及あり。
  - 自然言語生成への対応が期待されている。
  - 理由として「自然言語でのプロンプト理解が大幅に向上する可能性」。

- **krea2**
  - アニメイラストのベースモデル時点での出力品質が高く、背景の描き込みが強い。
  - リアル系・ファンタジー・古いエロ本風・電車のロングシートなどのテストに使われている。
  - 動画サムネ化やLoRA投稿時の表示問題との関連で言及。
  - 理由として「アニメイラストがしっかり出せる」「背景まで綺麗に描き込める」「普段やらない絵柄のテストに適している」点が挙げられている。

**Qwenシリーズ関連**：該当なし。
