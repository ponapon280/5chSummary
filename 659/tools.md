# 元の本文
**生成AI関連ツール レポート（変化・新規中心）**

### 1. Forge Neo（前回比で位置づけ変化）
前回までComfyUIが主流だったワークフロー環境において、**Forge Neo**が新たに有力な選択肢として複数言及された。主な変化点は以下の通り：

- **reforgeとの差別化**：Forge NeoはZ-Image turboやQwen、Animaなど最近のモデルへの対応が優位で、「新モデル対応では上位互換」との評価が定着しつつある。一方、reforgeは既存拡張機能との互換性を重視するユーザー向けの位置づけに分かれ始めた。
- **注意点の新出**：生成時間が徐々に伸びる現象が報告され、長期運用時の監視が必要とされるようになった。

初心者向けUIの使いやすさやSageAttention2/FlashAttentionのデフォルト最適化も相まって、ComfyUIからの移行検討材料として新たに浮上している。

### 2. 新規登場・注目されたツール
以下のツールは今回の抽出で初めて具体的な言及・評価が見られたもの：

- **Animerge**  
  Animaモデルをレイヤー単位でマージ・分析できる総合ツールとして登場。階層ごとの細かい制御が可能で、多言語化などの拡張作業に適している点が評価されている。ただしリポジトリの関数名起因のハルシネーション対策（gettext使用など）が必要という指摘も同時に出ている。

- **Unsloth Studio / LMStudio + llama-serverの組み合わせ**  
  12GB VRAM環境（RTX 4070/5070など）でGemma系モデルを量子化ロードしながらAnima生成を行う用途で新たに注目。LMStudioのGUIでモデル管理し、llama-serverのオプション調整で速度を確保する運用例が共有された。ComfyUIと交互に使用する「毎回イジェクト」運用も報告されている。

- **Hotgen Video V5**  
  NSFW対応を売りにした動画生成ツールとして初登場。seedance2.0並みの性能を謳い、クローズドモデル蒸留の可能性についても議論されている。

- **画像管理ツール群の再検討**  
  ComfyUI-Image-Browserの重さに対する不満から、**Infinite Image Browser（スタンドアロン版）**、**smartgallery**、**DiffusionToolkit**が代替候補として新たに挙げられた。Eagleからの移行先としても検討されている。

### 3. 継続的に変化が見られるツール
- **IdeogramHelper系**：Kijai氏版とは別の「アプリっぽいUI」の新ノードが登場し、扱いやすさが指摘された。
- **leco学習UI**：ADDifT（2枚の画像差分のみを学習する機能）の実装完了が報告され、差分学習需要に対応した動きが見られた。
- **Qwen（非画像生成用途）**：ローカルLLMとしてのプロンプト生成・コーディング・小説執筆用途が新たに複数確認され、Gemmaとの比較で「定型タスク・日本語安定性」で優位との声が出ている。

### まとめ
前回までのComfyUI中心の流れに対し、**Forge Neo**の台頭と**Animerge**・**Unsloth Studio**などの新ツール登場により、VRAM制約下での柔軟な運用やモデル特化のマージ・学習環境が多様化している。画像管理ツールも実用的な代替選択肢が増加した点が今回の特徴である。
