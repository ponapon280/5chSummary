# 元の本文
**なんJ（5ch）AI画像生成スレッド 統合レポート**

### 概要
このレポートは、5chなんJ板のAI画像生成スレッド（主にComfyUIユーザー中心）の複数ログを統合・要約したものです。テーマは**ローカル環境でのアニメ調画像生成**（特にNSFW寄り）で、**Anima**モデルが圧倒的な支持を集めています。技術的なワークフロー最適化・モデル比較・プロンプトTipsが活発に議論され、エロ生成やハードウェア話、下ネタも交じった典型的ななんJスレの雰囲気です。全体として「とにかく速く綺麗に生成したい」という実用志向が強く、Illustrious（SDXL）からの移行組やPony/RealisユーザーもAnimaへシフトしています。

### 主要モデルと評価
- **Anima（Base / Aesthetic v1.1 / Turboなど）**  
  現役最強クラスとしてスレの中心。顔崩れが少なくFace detailer不要で生成時間短縮、キャラLoRA成功率が高く「一発で綺麗」「ポン出し最強」と高評価。Turbo LoRA併用でさらに高速化可能（絵柄変化は最小限）。edit機能（服装・髪色変更など）は強いが、大規模変更（キャラ丸ごと置き換え・大幅ポーズ変更）は苦手。自然言語プロンプト＋Danbooruタグの併用が強みで、Aesthetic系ではscoreタグ不要という情報も。リアスやPonyからの移行者が多く、「絵柄劣化が少ない」と好評。一部で「上位互換が欲しい」という声もあり、足の指（6本）や横向き脚の描写に弱点あり。

- **Illustrious（SDXL）**  
  旧世代モデルとして「もう要らん」「肥やし」との声多数。Animaへの完全移行が進んでいます。高解像度（1536pxネイティブ）対応の強みはあったが、現在はAnimaに取って代わられつつあります。特定のマニアックLoRAはまだIllustrious依存のユーザーも。

- **Krea2**  
  実写寄り・審美重視のモデルで、Animaとは用途が異なります。Anima単体で物足りない場合の併用例あり。量子化（INT4）時にエラー（normalized_shape関連）が発生しやすく、ComfyUI最新版やcomfy-kitchen 0.2.18以上が必要と指摘されています。

- **その他のモデル**  
  DaSiWa（一貫性高めだが動き控えめ）、WAI（画質・プロンプト理解で高評価）など。Geminiは知識量は評価されるが生成能力で不満の声あり。

### ワークフロー・技術的話題
- **画像保存ノード**：Save Image Plus系は不安定で、**Save Image Extended**を推奨する声多数。webp/pngからのワークフロー読み込みトラブルが頻発し、「pngで上げろ」「imgbb使え」といったアドバイスが繰り返されます。
- **量子化・高速化**：INT4（W4A4/ConvRot/adaround）やINT8の話題が非常に多い。「全部INT4にするとゴミになる」ため混合精度推奨。RTX 50シリーズはINT4非対応でINT8フォールバック。Torchバージョン（2.10 vs 2.13）、SageAttention、Tritonの効果は「環境による」。Krea2ワークフローの非効率さ（256x256から多段アップスケール）も批判。
- **プロンプトTips**：`BREAK`多用は避ける（特にAnima）。品質タグの扱い議論あり。自然言語とタグ併用が安定。Anima Edit LoRA＋自然言語指示（Change/Remove xx）が便利。
- **多段サンプラー vs 低ステップ**：速度比較や最適化議論活発。

### 新モデル「AnimeGen」
Wan2.2ベースの国産アニメ特化T2Vモデル（AIdeaLab開発、METI/NEDO補助金関連）。正式公開されたが、スレでは「国産アピールで補助金狙い」「Wan2.2のガワだけ」「音声なしで微妙」「12時間かかる」と冷めた反応が多数。期待値は低めで、画像生成派の主流はまだAnimaに満足しています。

### NSFW・生成文化・注意点
エロプロンプト、キャラLoRA、精液表現などの話題が非常に多い。同級生DF（ディープフェイク）作成は法律違反の警告多数。中高生のゲーミングPC普及による量産可能性を「やばい」と指摘する声も。実在企業ロゴ（ポカリスエットなど）の使用はリスク高く、架空ブランド推奨。NTR作品論ではストーリー性（カップル描写→情けない彼氏→関係継続）の必要性が語られます。

### その他の話題
- **ハードウェア**：RTX 4090/5090、メモリ増設（64GB×2）、Gen4 SSD中古、2TBドライブ増設など。低スペック勢への当たりも強い。
- **保存形式**：webp vs avif vs jxlの比較。
- **コミュニティ傾向**：Anima信者増加中。Turbo使用論争（肯定派 vs 否定派）で派閥化しやすい。夏休み勢増加への懸念や「AI中毒」自虐、既婚/童貞ネタも。

### 傾向・総評
Anima一強の空気が支配的で、**ComfyUI + Anima**によるローカルアニメ生成の最適化がメイン。ワークフロー共有は活発だがトラブル多め。技術寄りながらNSFW欲求と実用志向が強く、量子化やプロンプトの細かい議論が特徴です。新動画モデル（AnimeGen）への反応は冷ややかで、画像生成の完成度に満足する層が主流。スレは技術共有から下ネタへ移行しやすい典型的ななんJパターンです。
