**抽出されたツール関連話題（モデル関連は除外）**

### ComfyUI（comfy）関連
- **ポストプロセスワークフロー（WF）での使用**（>>861, >>868, >>933）
  - flux9bで線画・白黒生成 → 色量子化（imagequant）→ 60線トーン貼り付け → 線画補正という一連のWFをComfyUIで構築。
  - 理由：手動では手間のかかるスクリーントーン加工を自動化・再現性高く行うため。変換時間は1分程度。
- **ComfyUI-Cosmos-Referenceノードの使用**（>>919, >>933, >>936, >>947）
  - Anima Edit LoRAと組み合わせた領域指定・参照ベース編集。
  - 特徴：マスクなしで着替えが可能（ただし場所指定は不安定）。上半身/下半身のみ変更などは運次第。
  - 理由：従来のinpaintより「マスク不要で自然に差し替えやすい」点を評価。動かない部分の固定精度は緩めで、細かい改変・差分作成に向いている。

### Forge Neo / ParamGui関連
- **Forge Neoでのプロンプト記法対応**（>>855）
  - コロン・セミコロンを使った特殊プロンプト（例: left: / right:）がForge Neoでも通るかを確認。
- **ParamGuiの使用とプリセット**（>>872, >>877, >>882）
  - 最新のParamGuiを常用。
  - 1024学習時のプリセットも試したが、512の方が学習精度が高かった事例あり。
  - 理由：GUIの更新状況や学習解像度設定の利便性を重視。

### その他のツール・スクリプト言及
- **GUI / sdscriptの更新確認**（>>872）
  - ツールやGUI、sdscriptの更新状況を聞く文脈で登場。学習時の解像度問題と関連づけて議論。
- **edit / inpaint系機能の比較**（>>849, >>883, >>898, >>955）
  - Anima Editをi2iや従来inpaintと比較。
  - 利点として「マスクなしで表情・前髪を保ったまま変更可能」「疑似inpaint的に雑な色塗りで置き換えやすい」点が挙げられている。
  - 欠点として「元の色を拾いやすい」「大幅なキャラ差し替えは苦手」。
