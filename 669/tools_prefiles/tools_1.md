**抽出された「ツール」関連話題（ComfyUI系中心）**

以下、ログから**ツール**に関する記述のみを抽出しました。モデル（anima / krea2 / illustrious など）の話は除外しています。

### 1. Save Image系カスタムノード（ワークフロー埋め込み・保存ツール）
- **audioscavenger/save-image-extended-comfyui**（Save Image Extended）
  - 現在最も推奨されている保存ノード。
  - webp/avif両対応で、メタデータとワークフローをしっかり保存できる。
  - インストールしていないComfyUIにwebpをドラッグ&ドロップしても「未インストールのカスタムノード」としてワークフローが読み込める点が便利。
  - Save Image PlusやSaveImage-PPで発生していた「後から読み込めなくなる」問題を回避できる。

- **comifyui-saveimage-plus (ID 1516)**
  - 以前は使っていたが、生成したwebpのワークフローが後から読み込めなくなる不具合があった。
  - Save Image Extendedに移行した人が多数。

- **ComfyUI-SaveImage-PP（ID4117）**
  - Save Image Plusの改良版と謳っていたが、同様にワークフロー読み込みに問題が出たため不評。

- **ComfyUI-Custom-Scripts**
  - ワークフロー入りのワークフロー画像を出力する際に必要になるノード。

### 2. ワークフロー読み込み関連のトラブルと対処
- Save Image Plusを使っていてもwebpのワークフローが読めないケースが複数報告されており、「自分で生成した画像でないと読めない」「ノードが足りないと読めない」などの現象が発生。
- 対処として「pngで出力して上げる」「imgbb.comに上げる」「Save Image Extendedに乗り換える」などの提案がされている。

### 3. その他のComfyUI関連ツール・機能
- **NoofyUI**
  - ComfyUIの中身をそのままに、インターフェースだけ見やすくしたツール。
  - 「配線工事せずに済む」という触れ込みだが、実際はComfyUIのワークフローを読み込んで使う形になるため、根本的に配線が不要になるわけではないという指摘あり。

- **app mode**
  - ComfyUIの機能の一つとして言及。「app modeを使えばいい」という簡易的な提案。

- **ComfyUIのターミナル（コンソール）**
  - エラーログを確認するためにターミナルを開く方法についての質問と回答あり（左サイドバーから開ける）。

### まとめ（ツールが選ばれている主な理由）
- **Save Image Extended**が現在最も支持されている理由：
  - ワークフローとメタデータの保存信頼性が高い
  - ノード未インストール環境でもワークフローを読み込める互換性の良さ
  - webp/avif両対応で実用性が高い

- Save Image Plus系は「後からワークフローが読めなくなる」問題で敬遠されている。
