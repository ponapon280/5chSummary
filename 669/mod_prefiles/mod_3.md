**抽出結果**

### 1. **illustrious（リアス / イラストリアス）**
- **出現箇所**: 
  - 440, 442, 453, 475, 540, 541, 543, 552, 577, 578, 587
- **言及内容と理由**:
  - 440: 現在メインで使用中。「幼い少女のちょっと固そうな尻」を出したいが、丸くふっくらした尻ばかり生成されてしまうため試行錯誤中。
  - 442, 453: 長い自然文プロンプトを入れても効果が薄く、尻の形状制御が難しい。**illustriousでは限界があるかも**と疑念を持っている。
  - 540, 543, 577, 578, 587: 水面に浮かぶエモい構図や、膝を曲げずに脱力した浮き方をさせようとすると、モデルが勝手に膝を曲げてしまう癖が強く、**アスペクト比や追加制御（CN/ノード）なしでは特定のポーズが非常に制御しにくい**。
  - 全体として「**特定の体型・ポーズの再現性**」で苦戦しているモデルとして扱われている。

### 2. **anima**
- **出現箇所**: 
  - 493, 494, 541, 577, 602, 609, 613, 621
- **言及内容と理由**:
  - 493-494: 7月に新モデルを4つリリース。「**まともなのはターボとbだけ**」と評価。
  - 541, 577: リアスで難しかった水面浮きや特定の比率・乳首出しのポーズが、**animaでは比較的出しやすい**（特に577で「Animaで出せたのはおおと思った」とポジティブ）。
  - 602, 609, 613: ライセンス（生成物販売禁止など）の話や、**Anima Edit**（i2i専用LoRA的な機能）として言及。参照画像を使って手直しができる点が評価されている。
  - 621: animaで生成した画像をスクリーントーン加工して使用。

### 3. **NovelAI（NAI）**
- **出現箇所**: 
  - 471, 474, 479, 483
- **言及内容と理由**:
  - 471, 474: v5のリリース時期について。「テスト始まってるし8月か遅くても9月には来る」との憶測。
  - 479, 483: テスターの質が低い（現代アートや驚き屋絵ばかりプレビューに出してくる、細かいフィードバックができない）と不満。**NAIで実用的なゲーエン用途が永遠に来ないのでは**という懸念。

### 4. **Wan / LTX**
- **出現箇所**: 
  - 498
- **言及内容と理由**:
  - 498: 1枚絵からドット絵アニメーションを作るブラウザアプリの話の中で、「**wanやLTX、ControlNetなんかでも一貫性のあるアニメーションは難しかった**」ため、AIを使わずアルゴリズムだけで実装したと説明。**一貫性重視でAIアニメ生成の代替手段**として言及。

**抽出されたモデル関連話題（除外対象除く）**

- **gptlive**  
  テキストチャット/Live両方で言及多数。API経由で喋り方・トーン調整が可能という点が評価されており、「人間らしい会話感が強くなってきた」「初期は歯向かってきたが改善が見込める」といった声あり。API向け調整の柔軟性が選ばれている理由として挙げられている。

- **Abliterated系LLM（特にGGUF）**  
  「LLMのAbliteratedを使いたい」という質問に対し、具体的に **huihui-ai/Huihui-Kimi-K2.6-abliterated-GGUF** が推奨されている。ローカルLLM用途での選択肢として話題に上がっている。

- **PiD1.5**  
  公式サンプルの挙動について「隠れたディテールや意味を拾うのは必ずしも正解じゃない」「滲みや破綻が気になる」といった指摘あり。細部解釈の特性が議論されている。

- **wai**  
  「waiいいよね、俺も既存とは別モデルと思ってwai使ってるわ」という言及あり。waifu系とは別枠のモデルとして扱われている点が特徴。

- **comfy-kitchen / kitchen**  
  ComfyUI関連のカスタムノード/環境として複数回登場。更新方法（pip install --upgrade）やforge neoとの併用について触れられているが、生成モデル本体ではなくツール寄りの話題。

（NAI v5、Anima、illustrious、FLUX、Wan、LTXなどは除外対象のため除外）
