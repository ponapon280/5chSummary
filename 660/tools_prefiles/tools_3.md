**ComfyUI（主にワークフロー・ノード関連）**
- Regional Controlnet / Anima LLLite Regional Controlnet の導入・テスト  
  理由：領域ごとにプロンプトを分けて制御したい場合に有効。t2i段階で領域適用強度やendを調整でき、inpaintより一貫性を保ちやすい。マスクの再利用も可能で、複数色指定やAttention Coupleとの併用例あり。色を同じ領域で複数回使う応用も確認された。
- アップスケール手法（潜在拡大＋i2i、ultimate sd upscale、モデル使用拡大など）の組み合わせ  
  理由：潜在拡大だけだとブレるためi2i必須。書き込み量を増やしつつ破綻を抑えたい、または元のディテールを維持したまま拡大したい場合に使い分ける。Anima使用時の高解像度出力フローとして3（モデル拡大＋低ノイズi2i）が現実的とされる。
- FaceDetailer / Detailerの活用  
  理由：生成後の顔修正や部分的な品質向上に使用。アップスケール後の仕上げ工程として組み合わせる例が多い。
- Scail2のワークフロー（SAM3使用のポーズ検出）  
  理由：無印版のvitposeよりリファレンスの崩れが少なく、動きの再現性が高い。1分長尺動画生成時の音ズレ対策と併せて検証されている。
- Lightning LoRAの設定（Q8 GGUF推奨）  
  理由：Q4 GGUFだとピンボケが発生するため、Q8に切り替えることで安定。smoothmix v5.0ワークフローでのピンボケ解消に寄与。
