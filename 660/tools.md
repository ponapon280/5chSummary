# 元の本文
**生成AI関連ツール レポート**

前回・前々回の抽出結果と比較し、**今回新たに登場したツール**および**状況に変化が見られたツール**を中心にまとめました。変化のないツール（ComfyUIの基本的な言及など）の繰り返しは省略しています。

### 今回新たに登場したツール

**PixAI（Webサービス）**  
ブラウザ/スマホのみで手軽に利用できる点が評価され、特にエロ生成や日本語対応の利便性からローカル環境との比較で言及されました。ローカル勢と異なりUIがシンプルで、基本無料で利用可能な点が選択理由として挙げられています。

**nano-banana（Nanobanana2）**  
Geminiの画像編集機能と連携するバックエンドツールとして新たに登場。画像生成・編集用途で利用されており、無料枠や混雑時の挙動についても言及されています。

**ffmpeg**  
ComfyUIの出力後の後処理（GIF変換など）で言及されたツール。新規登場で、生成後の簡易的な動画・画像変換用途として位置づけられています。

**NVIDIA Studio Driver（610.62）**  
ComfyUIおよびllama.cpp向けの最適化が施されたドライバとして新たに抽出。マルチGPU環境での安定性向上を目的に言及されています。

### 状況に変化が見られたツール

**Forge Neo / reforge neo**  
前回までの「軽快さ・Animaとの親和性」という評価に加え、今回はBREAK構文などの独自機能が利用可能である点や、モデル切り替えの容易さが新たに強調されました。ComfyUIとの併用・切り替えが便利という実用面での言及が増えています。また、プロンプト反映バグの報告とアップデートによる解決といった運用上の変化も確認されました。

**sd-scripts / kohya-ss**  
前回は主に学習スクリプトとしての一般的な言及でしたが、今回はQwen-Image VAEをAnima向けに2D最適化するPRの内容が具体的に取り上げられ、省メモリ・高速化効果が検証された点が新たに追加されています。AnimaでのLoRA学習における実用性が向上した形で言及されています。

**Regional-Controlnet（Anima-LLLite-Regional-Controlnet含む）**  
前回は主に領域指定制御の有効性でしたが、今回は漫画のコマ割りレイアウト制御やAttention Coupleとの併用例、マスク再利用の柔軟性など、応用範囲が広がった形で言及されています。t2i段階での領域適用強度調整が可能な点が改めて評価されています。

### その他の補足
- **LM Studio**：前回と同様にローカルLLM実行ツールとして継続的に登場していますが、Gemma4との組み合わせによるComfyUI生成画像のチャット転用といった新しい用途が追加されています。
- **sageAttention**：初出時と変わらず、ideogram4使用時の速度向上を理由に導入されていますが、今回は具体的な速度向上率（33%程度）の報告が加わっています。

以上が、今回の抽出結果で特に注目すべき変化点および新規ツールです。
