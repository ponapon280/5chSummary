**抽出結果：指定モデルに関する話題（理由含む）**

ログから該当モデル（NovelAI/NAI、illustrious/リアス/ill/IL、FLUX、Wan、Qwen-Image関連、anima、Z-Image/ZIT/ZIE、LTX/LTX-2.3）の言及をすべて抽出しました。特に「選ばれている理由・利点・欠点・用途」に該当する部分を強調してまとめます。Krea2などリスト外のモデルは除外しています。

### NovelAI (NAI)
- NAIの略だが「NVA部」と呼ばれる理由：V1当時の反AI活発期＋顧客流入急増に伴う検索避け（>>471, >>590）。
- テスター募集：NovelAI Diffusion V5のテスト開始間近。徹底テスト＋誠実なフィードバックを求める（Twitter/X or Pixivフォロワー1万人以上＋Discord必須）。過去になかよし驚き屋テスターで失敗モデルを出して叩かれた経緯あり（>>492, >>497, >>502, >>514, >>516, >>519, >>612, >>641）。
- 選ばれる/期待される理由：
  - 有料だがそれなりの品質。ローカル（ハード揃えれば実質無料で高画質二次・フォト三次）と比較されるが、SDXLとNAI4並走期は課金していた（>>532, >>533）。
  - V5で背景の正確さ、構図・ポーズ制御が効くか気になる（>>519）。
  - v4/v3時代のように下着・性器の品質向上＋タグミックス継続を希望。ミックスできないとanimaと変わらない（>>613）。
  - リリース間隔参考：v4.5は宣言→テスト→Curated（2週間後）→Full（さらに3週間後）。学習中発言から最長数ヶ月・最短1ヶ月程度の可能性（>>616）。
- その他：有料確認（最初30枚無料）、過去応募落選経験、キャラ1枚でデザイン再現機能はanima等でまだ真似できない（>>556, >>593, >>637, >>639）。

### illustrious（イラストリアス, リアス, ill, IL）
- リアス登場以降、シチュエーション毎のLoRA不要になり、danbooru語さえ合っていれば望みのシチュを出せるようになった（>>578）。
- 具体使用例：モデルとして`waiIllustriousSDXL_v170`を使用。ペニスの色を白っぽく固定・サイズを小さく安定化させる方法を質問（>>635）。
- **選ばれる理由**：イラスト系で状況再現の自由度が大幅向上（danbooruタグだけで制御可能）。

### FLUX
- デカい系モデルの一つとして言及。すでにfp8化済みのものはint8 convrot変換が無理なので、元モデルを確保し直す必要あり（Anima・ZITは変換確認済みだがFlux/Qwen/Krea2系は再確保）（>>487）。
- **選ばれる/注意点の理由**：大型モデルとして扱われ、量子化（int8など）時に元ファイル管理が必要。

### Wan
- WANだと多少時間かかるが、**元絵の雰囲気を崩さず動画生成できる**。一方LTX2.3では元絵の雰囲気を崩さずに動画生成できない。WANで動画生成＋LTX Directorで声入れが一番良いのでは？（>>523）。
- **選ばれる理由**：動画生成時の元絵雰囲気保持性能がLTXより優れる（時間はかかるが品質優先）。

### Qwen-Image（Qwen系含む）
- Qwen系モデル（zen-bear-v3など）：パッと見GPT-image-2に似ており、**文字出力の精度はかなり高い**。レイアウトのセンスは（明確指示なしでは）しょぼい（>>472）。
- int8_convrot版QwenImageEdit2511：DL可能。RTX4070Ti＋Lightning-4steps LoRAでfp8版より約0.6s/it高速化（質の見極めはこれから）（>>591, >>594, >>598）。
- 自分でint8変換済み（z_image turboと一緒に）。Qwen3VL 8B NSFWを画像→プロンプト取得に使用（検閲回避にシステムプロンプト必要）（>>487, >>557, >>608）。
- **選ばれる理由**：文字精度の高さ、Editモデルとしての実用性、int8化による速度向上。キャプション/プロンプト生成用途でも活用。

### anima
- Anima・ZITは自分でint8 convrot変換確認済み。WAI-ANIMAのINT8 Convrot版も出ている（>>487, >>592）。
- anima baseのLoRA学習は**小物の学習精度がかなり良い**。自室家具を学習素材に加えたら面影が出た（SDXLでは失敗したがanimaなら成功。感情が揺れるレベル）。自撮りちんぽしゃぶらせ系にも応用可能性（>>494, >>495, >>496）。
- ANIMAでテカりすぎることがあったのでshinyスライダー（マイナス想定、dim4alpha4）を作成中（>>490）。
- anima base1もint8 convrotした方が良いか？（>>515）。
- ComfyUIでは使えずWebUI Forge Neoを入れたがエラー多発・UIが見にくくアンインスコしてComfyに戻った（>>575）。
- NAIのキャラ1枚デザイン再現機能はまだanimaで真似できない。タグミックスできないとanimaと変わらない（>>593, >>613）。
- そろそろanima base1.2が出るか期待。MoeFussion 3.0 Anima class使用例：顔・絵柄はモデル由来でキレイめ（SFW向き）だが、cum-glisten/cum-coveredなどの汁表現が苦手（>>615, >>627）。
- Animaでも特定体勢で手の形が左右逆になりがち。kサンプラー即貼りも「アニマだもの」と許容（>>631, >>652）。
- **選ばれる理由**：小物/家具LoRA学習精度の高さ、int8対応による軽量化・高速化、ベースとしての使いやすさ。欠点として汁表現弱さ・手の破綻・一部機能不足（NAI比較）。

### Z-Image（Z-Image Turbo, ZIT, ZIE）
- Anima・ZITあたりは自分でint8 convrot変換確認済み。ZITは5GBくらい縮んで「しゅごい革命」（>>487）。
- z_image turboも自分で変換。特に**生成速度が速くなる**（>>608）。
- **選ばれる理由**：int8 convrotによる大幅なサイズ縮小（約5GB）＋生成速度向上。品質ほぼ落ちない高速化が評価。

### LTX（LTX-2.3, ltx2.3）
- LTX2.3では**元絵の雰囲気を崩さずに動画生成できない**（WANなら可能）。WANで動画生成してLTX Directorで声を入れるのが一番良いのでは？（>>523）。
- ltxのdmdってやつは3段サンプラーより早いのか？（>>576）。
- **選ばれる/比較理由**：雰囲気保持ではWANに劣るが、Director機能で音声付け用途として組み合わせ推奨。dmdサンプラーの速度が関心事。

### 横断的な話題（複数モデル関連）
- int8 convrotの恩恵：Anima・ZIT・Qwen（Edit含む）・z_image turboなどで変換・使用報告多数。fp8より速度・精度上、古いグラボでも恩恵大。ファイルサイズ問題なければ最高評価（>>487, >>515, >>560-574, >>581, >>584, >>592, >>598, >>605, >>608など）。
- 全体としてローカル勢がこれらのモデルを高速化・LoRA学習・動画用途で積極活用。NAIは有料サービスとしてV5期待、ローカルはanima/illustrious/Z-Image/Qwen/Wan/LTXの組み合わせが主流。

**抽出結果：生成AI「モデル」関連の話題（指定除外対象を除く）**

除外対象（NovelAI/NAI、illustrious系、FLUX、Wan、Qwen-Image、anima、Z-Image/ZIT/ZIE、LTX系）への言及はすべて無視し、それ以外のモデルに関する記述のみを抽出。特に「選ばれている理由・特徴・比較」が明確なものを優先。

### 1. Krea2（および派生/関連Checkpoint・Edit）
最も頻繁に言及された非除外モデル。Checkpoint選択、NSFW適性、LoRA相性、Edit機能が中心。

- **Checkpoint選択の議論**（457, 458, 479）  
  - 公式Modelでよいか、NSFW用途ではRedCraft / Yogi / Moodyの方が良いかという質問。  
  - **RedCraftの理由・特徴**：指示していないのに勝手にペニスを描くため使いづらい面がある一方、「構図の自由度は高い」と評価。  
  - Krea2自体は「アジア人も得意」なため、ジャパング（日本人向け）LoRAの効きが分かりにくいという指摘。myjs LoRA追加を待つ声あり。

- **LoRA使用時の問題と対策**（462, 505, 549, 553）  
  - LoRAを噛ませると激重くなる症状あり。CUDAをcu124→cu130に上げたら大幅改善（15分→3分程度）した報告。  
  - **Krea2 Engineer**：エンコーダーに癖があり、NSFW系LoRAを使うと「どうやっても分厚い唇の白人顔が混ざってきて調整がきかない」。  
  - Engineerの役割をAIに尋ねたところ「しょぼいプロンプトをいい感じに盛ってくれるやつ」と説明され、使い分けが必要との認識。

- **Editモデル化**（601, 606, 619）  
  - Krea2をEditモデルにするLoRAが共有され、試用報告あり。  
  - **選ばれる理由**：「編集後も絵柄がそんなに変わらない」点が好評。バニーガール化やカジノ背景変更などの例が挙げられている。

- その他：色温度スライダーLoRAとの組み合わせで勝手に脱ぎ始める現象、Forge Neo環境での使用など。

### 2. zen-bear-v3（Qwen系）
- **言及**（472）  
  LMArenaで発見されたモデル。  
  - **特徴・理由**：パッと見GPT-image-2によく似ており、「文字出力の精度はかなり高い」。一方で「レイアウトのセンスは（少なくとも明確な指示なしでは）しょぼい」。

### 3. Gemma 4（および関連VLM/LLM用途）
画像分析・キャプショニング・物語生成での使用が中心。ローカルLLMとしての評価。

- **エロ画像分析・文章生成での課題**（527, 534, 536, 537, 540, 541, 544）  
  - がっつりセックス画像を読ませても「直接的な表現が少ない柔らかなもん」になり、体位などが一切出ない。  
  - **理由・性質**：元々文章がテンプレに陥りがちで表現が婉曲。画像認識も「あっさりぼんやり気味」でプロットが凡庸になりやすい。  
  - 手書きプロットを与えると良い出力になるが、画像からそれを引き出すには工夫が必要。  
  - エロ特化では「ガッツリDanbooruを学習しているTaggerモデルのほうがLLMより賢い」との指摘（世界トップ研究者よりユーザーの方がエロに詳しい、という比喩）。  
  - 対策案：システムプロンプトで「体位や姿勢を明確に識別して絶対に書け」と指示、またはエロ単語辞書を与える、Tagger出力をフィードして「これらのキーワードを使ってdescribeせよ」と指示する手法が有効。

- **全体評価**（536）：以前のモデルと比べたら「夢みたいな性能」で、不満を言うのは贅沢、という認識。

- 関連：Qwen3VL 8B NSFWもキャプション用途で言及されるが、拒否や長文吐出がありシステムプロンプト必須。joycaptionの方が再現度が高かったという比較あり。

### 4. fable（fableたん / fable5など）
- **言及**（524, 531, 562, 621）  
  - サブスク期間終了が迫り「離れたくない」「むっちゃ頭良い。こいつ無しの生活は考えられん」と強い愛着。  
  - 国産fable級モデルを作れという冗談も。  
  - deepseek-v4-proとの比較で「FlappyBirdクローンでfable5の525分の1のコスト」とされ、コスト面で劣る。

### 5. deepseek（deepseek API / deepseek-v4-pro）
- **言及**（613, 617, 618, 621, 623, 633）  
  - Gemma4から移行した例あり。  
  - **選ばれる理由**：  
    - ローカルGemma4より良いと感じる。  
    - 他クラウドAPI系と比べ性能は最高ではないが、「脱獄が簡単で手軽に安くエッチできる」。  
    - コストが非常に低い（fable5比で大幅安）。混雑時2倍値上げでも許容範囲。  
    - 課金は事前にドルを買ってその範囲で使用する方式（PayPal可、使いすぎ請求なしで安全）。  
  - openrouter経由の方が公式契約より良いのでは、という提案も。

### その他軽微なモデル・関連言及
- **Oppai Oracle v1.1**：最新Taggerとして使用。出力タグが多すぎてLLMがエラーを吐く問題があり、しきい値調整やコンテキスト長増加で対応。
- **joycaption**：キャプション特化で再現度が高かったとの比較。
- **int8 convrot**関連：技術自体はモデルではないが、各種モデル（非除外含む）への適用で「fp8より精度・速度が上」「品質ほぼ落ちない」「地球にやさしい」と高評価され、Forge NeoやComfyUIでの採用が進んでいる文脈で登場。古いグラボ（RTX30以前）で特に恩恵、RTX40/50でも有効という意見が多数。
- **MoeFussion 3.0 Anima class**などはanima関連のため除外。
- waiIllustriousSDXL_v170はillustrious系のため除外。

**全体的な傾向**  
非除外モデルでは**Krea2**が画像生成の実運用議論の中心（NSFW適性・アジア人表現・Editの安定性で選ばれている）。LLM/VLM系では**Gemma 4**の婉曲さ・認識のぼんやりさを補う工夫と、**deepseek**の「安くて簡単にエロ生成できる」実用性が強調されている。fableは性能・愛着面で支持されているがコスト比較でdeepseekが優位に語られる。
