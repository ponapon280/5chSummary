## 2. 主要トピック別サマリー

### 2.1 LoRA学習設定とステップ数の理解

**論点：** 「1000ステップでOK」「3000ステップ」など個人差が大きく、バッチサイズ・勾配累積・解像度・学習枚数・LoRA種別（構図/キャラ/絵柄）で条件が違うため、ステップ数の単純比較は無意味。

| 投稿者の実践例 | 内容 |
|----------------|------|
| 画風LoRA（リアス） | 800枚、プリセット通り → 完成度約8割。残り2割はAnimaの良い部分が混ざり好印象 |
| キャラ(+画風) LoRA（Anima） | Prodigy、1024、バッチ10、約800ステップ → 5090で約11.5時間 |
| Krea2 ドット絵LoRA | ai-toolkit、1024、2000ステップ、100枚、batch1 → 約2時間、Yellow Buzz 1,475 |
| Krea2 NSFW LoRA | AdamW_adv、cosine_with_min_lr、lr=0.0001、dim/alpha=32、1024×1024、30枚、20epoch |
| 画風LoRA（比較） | リアス10時間 / Anima20時間（妥協ラインを下げない設定） |

**合意点：**
- 正解のステップ数は存在しない。求める品質と妥協点は個人で試すしかない
- 解像度とバッチサイズの影響が特に大きい
- まず512解像度の高速プリセットで短時間検証 → 本学習、が推奨される流れ

### 2.2 VRAM漏れ・GPU電力低下問題（>>38〜61）

**症状：**  
ai-toolkitでLoRA作成時、GPU使用率は100%近いが消費電力が低く、進行が極端に遅い（RTX 5080 / メインメモリ96GB環境）。

**診断と対応：**

| 原因 | 説明 | 対策 |
|------|------|------|
| VRAM溢れ → 共有GPUメモリ（メインメモリ）へのオフロード | メインメモリは遅いため激遅化 | タスクマネージャで「共有GPUメモリ」を確認。通常0.2〜0.3GB程度。増えていれば漏れ |
| GPUコア占有のみでCUDA/テンソルコアが動いていない | 使用率100%でも電力・温度が低い | VRAM余裕確保、batchサイズ低下、解像度512化 |
| カツカツ設定での学習 | 16GB VRAMは動画LoRAや大モデルで特に厳しい | 枚数削減、解像度・batch調整 |

**実務Tips：**
- 専用GPUメモリに数GB余裕があれば基本問題なし
- スワップ発生まで行くと本番級の遅延
- SSDへの溢れはコスト的にも避けたい、という認識が共有

### 2.3 ハードウェアトラブル（電源が入らない）

| 状況 | 対応・考察 |
|------|------------|
| 昨日まで正常 → 朝電源が入らない | コンセント抜いて放置（数時間〜1日）で復旧した事例あり |
| 原因候補 | 保護回路のラッチ、電源故障、マザボのCMOS電池切れ、電源ボタン接触不良 |
| 再発時の推奨 | 電源 or マザーボード交換を視野に。焼損の目視確認。最小構成テスト。AM5+ASRock構成ではCPU故障率にも注意 |
| 補足 | 冬場以外の起動不能は死に近い可能性が高い、との警告あり |

### 2.4 モデル比較：Krea2 vs Anima vs その他

| 観点 | 内容 |
|------|------|
| Krea2 | 話題の中心。プロンプトの利きは良いという声。LoRA前提の運用感。civitaiでトレーニング対応開始。FFTは12B級で個人環境は厳しい（5090×2でブロック分割学習の実験例） |
| Anima | エロ性能は依然として強いとの認識。LoRA作成時間が長いという印象と、設定次第で短縮可能という反論が併存 |
| Illustrious（リアス） | 画風LoRAが比較的短時間で終わる事例。使い方を忘れつつある、との声 |
| 比較の空気 | 「Krea一色になるとAnimaは終わった感」→ しかし上位互換と言っている人はおらず、超えている部分もある程度。新もの好き傾向が強いスレ文化 |

**int8 conv関連：**  
Comfy公式（kijai氏push）のスクリプトでINT8_convrot化が可能。Animaでは生成時間ほぼ変わらずファイルサイズ半減、という報告。40系以降はfp8で十分な場合も。

## 3. 技術ナレッジの抽出（実務メモ）

### LoRA学習で頻出の失敗パターンと対策

1. **VRAM漏れで激遅** → タスクマネージャで共有GPUメモリ確認。batch/解像度/枚数を落とす  
2. **ステップ数信仰** → バッチ・枚数・解像度・LoRA種別をセットで比較する  
3. **一発本番** → 512高速プリセットで短時間試走してから本学習  
4. **Animaが遅いという固定観念** → 設定次第で短縮可能。リアスが速い/Animaが遅いはサンプリングバイアスの指摘あり  

### Krea2関連の実務メモ

- civitaiでトレーニング対応開始  
- FFTは個人では重い（12B、ブロック分割＋マルチGPUの実験例：3000枚で1epoch約4時間）  
- RedRayz改良GUI + 対応sd-scriptsで5060Ti級でも現実的な時間でLoRA可能（条件付き）  
- INT8/fp8は環境（30系 vs 40系以降）で選択  

### PC起動不能の応急処置フロー

1. コンセント抜き放置（保護回路リセット）  
2. 電源ボタン長押し・強押し  
3. CMOS電池確認  
4. 最小構成 + 基板上パワースイッチ  
5. 電源 → マザボ → グラボ → CPU/DRAM の順で疑い

## 5. 結論・示唆

1. **本ログの価値**  
   LoRA学習の「ステップ数だけでは比較できない」という合意形成、VRAM漏れの実機診断手順、Krea2導入初期のプリセット・時間感、ローカルLLMとクラウドCodexの役割分担など、**再現可能な実務知**が濃い。

2. **技術トレンド**  
   Illustrious/Anima中心から **Krea2への関心期**。画質・プロンプト追従とマシンパワーが結びつく認識が強まり、16GB級では設定の工夫が必須。動画側の新モデル待ちの声もある。

3. **ツールエコシステム**  
   個人改造GUI・Animerge（DPO）・INT8変換など周辺が活発。一方で **GitHub中心の配布文化** と **即時バイナリ共有** の緊張が続いている。

4. **運用上の推奨（ログから導かれる）**  
   - 学習は「短時間プリセット検証 → 本番」  
   - 遅延時はまず共有GPUメモリと電力を見る  
   - モデルは単一絶対視せず、用途（エロ特化 / プロンプト精度 / 画風）で使い分け  
   - LLMは「設計はクラウド、実装・整理はローカル」のハイブリッドが現実的  

5. **注意**  
   ログには露骨な性的コンテンツおよび問題のある題材への言及が含まれる。レポートは会話の整理を目的とし、違法・有害な生成物の作成や利用を推奨するものではない。
