## 1. LoRA学習・変換ツール

### ai-toolkit
- LoRA作成に使用。以前は同じ設定でGPUをちゃんと使えていたが、今回はGPU使用率100%近いのに消費電力が低く進行が激遅になる不具合の相談あり（VRAMがメインメモリに漏れている疑い）。
- ちびたいWEBトレーニング経由でも `engine: "ai-toolkit"` として利用される。

### kohya_lora_gui / kohya_lora_param_gui（RedRayz改良版）
- RedRayz氏が **krea2対応の改良版** を公開。
- 使用例: CAME・3エポック・fp8_scaled、5060Ti(16GB)/128GBで約4時間でLoRA生成成功。
- プリセットあり（Anima用など）。**512×512高速プリセット＋p3版** だと1時間未満で終わり、出来も良いという理由で推奨。
- 解像度とバッチサイズが時間に最も影響する、という運用知見。
- ダークテーマ追加の要望・対応あり。
- 配布がGitHub外であることへの是非議論（MITライセンス上は出処明示で問題ないが、オープンソース精神・セキュリティ誤検出の観点でGitHub推奨の声）。
- 動作には **sd-scripts側の対応ファイル** も必要。

### ちびたいWEBトレーニング
- Krea2用LoRA焼成に使用。
- デフォルト設定・2000ステップ・約2時間で完了した例あり。
- パラメータ例: adamw8bit、dim/alpha 32、resolution 1024、batchSize 1、ecosystem "krea2" など。

### ComfyOrg / kijai の int8 conv 変換ツール
- **Comfy公式にkijai氏がpushしたスクリプト**で、通常チェックポイントを INT8_convrot 化できる（約1分）。
- 選ばれる理由: 公式／kijai製でないと安心できない、という心理的ハードルを満たすため。
- Animaでは数秒で変換完了、ファイルサイズ半減。生成時間はほぼ変わらず。
- 容量節約＝負荷・速度軽減につながるので標準化していくだろう、という見通し。

### Animerge
- 更新内容: **ADDifT学習のDPOモード**（SDPO・MaPO実装）。
- win画像（望む表現）とlose画像（嫌う表現）をセットし、学習モデルのバイアス経路を変更する差分学習。
- LECOに近いが、概念消去＋強化ではなく経路遮断・誘導に近い調整。
- 細かいニュアンス調整に有用、という評価。
- 導入: リポジトリを git clone → `setup_start.bat` 実行。

### negpip
- 導入に四苦八苦したが、ノードを色々繋いだら動作した、という報告。

## 2. ワークフロー・UI関連（Comfy系）

### krea2公式ワークフロー / ComfyUIノード
- 公式ワークフローで「enable Lora」をtrueにすると生成が進まなくなる問題。
- int8モデルは専用カスタムノードやライブラリ（独自テンソル操作のフック）が必要、という調査結果。
- 40系以降GPUならfp8版で十分、という代替案。
- 公式がkrea2専用ノードを要求するため、Anima用ワークフローを流用して最低限生成する運用例あり（`mu`パラメータは無視）。

## 4. コーディング／LLMツール（Qwenは画像生成以外）

### ChatGPT Plus / Codex / Pro
- 日常ではCodex使用量がほぼ余っていたが、**ゲーム開発を始めて枯渇**。
- Pro（約16800円）推奨の声。理由: リミットが大きい／GPT 5.5 Proでゲームデザインの壁打ちが捗る／ClaudeユーザーはほぼMaxプラン。
- 「Plus＋ローカルLLMで一本完成させてからPro検討」という慎重派意見。
- Codexの強み: 言ったことを一発でやる、GitHub連携。
- トークン節約ネタ: プロンプトを画像で渡して入力トークンを減らす（ただし入力側は元々安くすぐ対策されそう、という懐疑）。
- 5.6 Terraは同等性能でコスト低下予定、5.6 SOL近日ローンチ、という話題。

### LM Studio ＋ VS Code ＋ Qwen3.6-27B
- ローカルLLM運用例。
- **Gemma4は日本語優秀だがコーディングではQwen3.6より落ちる**、という比較。
- 運用方針: 日々のルーチン・細かい修正はローカル、大規模改修・新規はCodexに骨子を組ませ、引き継ぎ文書も用意。
- Codexに相談して作業向けシステムプロンプトを作ってもらう使い方。

### Ollama ＋ Qwen3.6
- 短コンテキストだと架空の変数を出すなど不安、長めコンテキストなら実用的。
- VRAM構成例: 5080(16GB)+3080(12GB) 計28GB。
- **役割分担の明確化**:
  - Qwenにやらせてよい: 関係ファイル選別、既存フロー要約、実在メソッド名抽出、仕様差分表、リスク候補列挙、Codex投げる前の論点整理
  - 危ない: そのまま貼れる擬似差分、最終実装指示の確定、「安全です」と断言、既存メソッド改造の具体案
- 結論: 壁打ち相手として運用。

### Gemini
- 二枚挿しGPU時のPCIレーン（×8が2本ないとボトルネック）について相談に使用。
