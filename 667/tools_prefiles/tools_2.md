以下、ログから**生成AI関連の「ツール」話題のみ**を抽出しました。  
（モデル本体の話題＝Anima / Krea2 / Z-Image / NAI / リアス / FLUX 等は除外。Qwenは画像生成以外のみ採用。理由があるものは強調）

## 1. ComfyUI（comfy）

- **ワークフロー作成方針**
  - ワークフローはJSONなので、AIに全部任せるより「自分で組み立て＋AI製カスタムノード」が現状ベスト（313, 295）
  - 来年くらいにはAIでワークフロー作成が普通にできるだろう（313）
  - 直接AIに一発で作らせるより、必要ノードを聞いたり他人のノードを参考に組み合わせる方が早い（260）
  - エラー参照・循環参照エラー・ライン接続ミス・ライブラリ不足の特定はAIが強い（260）

- **ComfyUI MCP**
  - ワークフロー作成をより正確に支援してくれるのでおすすめ（418）
  - IDEのAIエージェントに「Agent, please help me install Comfy MCP:」と唱えればインストールできる（418）
  - **理由**: WebUI単体だとキツい。IDE＋MCP環境の方が正確（418）

- **VRAM・起動オプション**
  - Krea2のLoRA使用時にVRAMが溢れる問題への対処として、起動batに  
    `--enable-dynamic-vram --novram` を追加する話（302, 305, 308, 311）
  - Dynamic VRAMはComfyUIでデフォルト有効になったが、環境によってはまだ明示が必要という報告（305, 308）
  - `--novram`だけだと遅すぎ、`--lowvram`だと溢れる、という挙動差（308）
  - 一方、VRAM12+RAM64/128ではオプション無しでも問題ない人もいる（312, 314）

- **GenerateTextノード（ネイティブ）**
  - さくっとテキスト生成するならComfyUIネイティブのGenerateTextノードが使える（407）
  - Krea/Ideogram等で使っているTEからもテキスト生成可能（407）
  - GGUF CLIP Loaderを繋げば動くのでは、という提案（417）
  - 画像入力（mmproj）までできるかは要確認（424）

- **その他**
  - 潜在アプスケが使える（431）
  - 背景画像は設定から変更可能（432→435）
  - 現状推奨環境: pytorch2.12+cu130以降（440）

## 2. WebUI / Forge Neo

- ChatGPTのWebUIだとカスタムノード作成は単機能ノード1つ程度が限界（255, 259）
- **理由**: 本格的なワークフロー・カスタムノード作成はCodexやIDE側の方が向いている（255, 418）
- Forge Neoのint8 ConvRot対応
  - 当初「使えない？」→ int8ブランチならほぼ問題なく使える（337, 415）
  - その後本体にマージされた（439）
  - よく分からなければマージ待ち推奨（353, 415）

## 3. Animerge（インストール／起動ツール）

- Pythonパッケージも同梱してほしい（246）
- 仮想環境ができれば後はstart.batだけで起動可能。torchバージョン等を合わせたい場合は手動（250）
- **インストール方法の注意（重要）**
  - ZIPから入れるとbatの改行コードがLFのままでエラーすることがある（252, 256）
  - **git cloneからのセットアップ推奨**（256）
  - 手動でCRLFに直せばZIPでも行けるが、バージョン管理もあるので基本はgit管理（256）
  - ZIPで成功した人もいるが、上記が原因だったケースあり（253, 257）
  - AIに聞くと「ZIPから入れろ」と言われがちなので、git推奨を書いておくべき（252）

## 5. ローカルLLM実行ツール（Ollama / LM Studio / llama.cpp 等）

| ツール | 評価・理由 |
|--------|------------|
| **Ollama** | 不評。ggufをmodelfileでcreateすると同一物をコピーしてSSDに書き込む仕様で非推奨（410）。llama.cppを隠していた・AIで書き換えただけ・自作版は遅い、と炎上（434）。APIは最初から有効（412） |
| **LM Studio** | Ollamaより楽。model searchで探してすぐDLできる（406）。API有効化手順が記事に無い場合あり（412）。最近はMTPや技術的限界で風向き変化の噂も（444）。実質llama.cppラッパーなのでキャッチアップが遅延しがち（447） |
| **llama.cpp / llama-server** | Ollamaより推奨する声。環境をAIに伝えればインストール方法・起動パラメータを教えてもらえる（430）。LM Studioより直接触る派も（436） |
| **lemonade** | 軽くて速いと聞いたが、システムプロンプト機能が無い（423） |

- 導入候補モデル例として Uncensored 系（Qwen3.5-9B-Uncensored等）をOllama/LM Studioで使う話（405）※画像生成以外のQwen利用
- ComfyUIのGenerateText経由でローカルTEを使う案もあり（407, 417, 424）

## 6. ChatGPT（チャッピー）/ Codex / その他クラウドLLM（コーディング・WF支援ツールとして）

- **ワークフロー作成**
  - チャッピーにWFを頼んだら焦らされた挙句実行してくれなかった例（248）
  - 指示が相槌になっていた・一括で頼んでいた・無料低品質モデルだった、などが失敗要因（270）
  - カスタムノードはチャットで作らせてzipで貰う運用も可能。正規表現＋スケジュール構文対応テキストエンコーダ程度ならCodex無しでも作れる（273）
  - ChatGPT ProならCodex無しでもjson渡せばjson作成まで可能。PlusのThinkingでも可（285）
  - エラーメッセージを貼って修正させる流れ（285）

- **Codex推しの理由**
  - 「みんながカスタムノード作らせてる」のはWebUI版ChatGPTではなくCodexだと思う（255）
  - コーディング・構造化データではIDE/Codex環境の方が向く（418）

- **運用Tips**
  - 画像生成ツールを勝手に起動してくるので「画像生成ツール起動禁止」を毎回入れる（266, 268, 284）
  - メモリやプロジェクト指示に書いても貫通することがあり、毎回言う必要あり（284）
  - ワークフローは「まとめて」ではなく「一個ずつ」聞く（270）
  - 理解負債を避けるため、WF全体を任せるのはやりすぎ。カスタムノード作成や修正方法を聞くくらいがちょうどいい（295）

- その他: Gemini、Claude、Fable（高度なコーディング用途・月額上位プラン）への使い分け話（263, 272, 276, 282, 287, 289）

## 7. LoRA学習・周辺ツール

- **ADDDifT**: 短時間で確実な効果の出るお手軽スライダーが作れる。強すぎる、という評価（249）※Anima用夕日効果もどきスライダー、5090で学習約1分
- **ai-toolkit**: LoRAはこれで焼く運用（306）
- **kohya_lora_gui_krea2 / sd-scripts（krea2対応版）**: 直接配布を避けGitHub公開。匿名性・安全性の担保として評価（387, 395）

## 9. Qwenシリーズ（画像生成以外のみ）

- ローカルLLMとして Qwen3.5-9B-Uncensored 系をOllama/LM Studio等で使う話（405）
- ComfyUIのTE（Qwen系含む）からGenerateTextでテキスト生成する使い方（407）
- ※Animaがなぜ1GB級Qwen言語モデルを選んだか、というアーキ議論（358, 392）は「モデル選定」寄りのため、ツール抽出からは除外気味に扱うのが適切

## 抽出のポイントまとめ（理由付きで選ばれているツール）

| ツール | 選ばれる／推される主な理由 |
|--------|---------------------------|
| **ComfyUI + MCP** | WFがJSONで扱いやすい／MCPでAI支援が正確／カスタムノードと組み合わせやすい |
| **git clone（Animerge）** | ZIPだとbatのLF改行でエラー。git管理＋バージョン管理が安全 |
| **Codex / IDE** | Web ChatGPTよりカスタムノード・コーディング向き |
| **LM Studio / llama.cpp** | Ollamaの不評（コピー書き込み・性能・炎上）を避け、導入のしやすさ or 本家寄りを取る |
| **nano-banana** | 一発での画面同期・再現性が高い |
| **ADDDifT** | 短時間で確実に効くスライダーが作れる |
| **SCAIL-2** | ダンス等の置き換えが比較的簡単 |
