**抽出結果**

### 主な話題モデルと選定理由

**anima（Anima / WAIanima / Hassaku Anima）**
- スレ内で最も多く言及されているモデル。
- **リアスとの比較で選ばれている理由**:
  - プロンプト理解力・忠実性がリアスより明らかに高い（特に長文・自然言語プロンプト）。
  - 75トークン制限やBREAK連打を意識しなくて良い。
  - キャラLoRA使用時の画風汚染を階層カットで抑えやすい。
- WAIanimaは「マンピー（男性器）の綺麗さ」で評価されている。
- Hassaku Animaは「早く試したい」との声あり（Civitai待機中）。

**リアス（illustrious / IL）**
- animaとセットで比較されることが非常に多い。
- animaに乗り換えた人が「画風の安定感」や「特定のテイスト」ではまだリアスの方が良いと感じるケースあり。
- 「animaのマージモデルを作ってる人はリアスに戻ればいいのでは？」という意見も出ているが、逆に「リアスにはもう戻れない」という声も。

**WAIanima**
- animaの派生として言及。
- base animaとの違いが分かりにくいという感想あり。
- 男性器のクオリティでWAI版が優位という評価。

### その他言及
- **Pony系**: ストレージ確保のために削除したという話のみ（積極的な使用話題なし）。
- **Ideogram 4.0**: 検閲の厳しさについて複数言及（animaとの組み合わせ使用例あり）。

**まとめ**  
このログでは**animaとリアスの比較**がほぼ全ての話題を占めており、特に「プロンプト理解力の向上」がanimaを選ぶ最大の理由として挙げられています。

**抽出された「モデル」関連話題**

### 1. Pony系モデル
- ストレージ容量確保のためPony系を全削除したという報告あり。
- 削除前にXYZ（推測：XYZプロット用LoRAやモデル）だけは残そうとしたが、Score系を入れ忘れて失敗したエピソードが語られている。
- 理由：単純に容量不足。特定の性能優位性についての言及はなし。

### 2. WAI（WAI Anime系）
- 「エロならWAI」という短い推奨投稿あり。
- Anima（除外対象）と比較される文脈で出ており、エロ生成での実用性を評価する声が見られた。
- 理由の言及は簡潔で「エロ向き」という点のみ。

### 3. Ideogram 4.0（ideogram4 / ideogram4.0）
- スレ内で最も言及が多いモデル。
- 主な話題：
  - 検閲（safety filter）の厳しさ（パンチラ程度でもブロックされる、JSONプロンプトでも挙動が不安定）。
  - JSON Prompt Builderの使用で回避しやすくなるという指摘。
  - 1ステップ目をスキップする手法で検閲を回避できる可能性がX上で話題になっているという情報共有。
  - Flux2 VAEを使用しているため、LatentをFlux系モデルへ引き継げるという技術的言及。
- 理由：自然言語での生成が強い一方で検閲がネックになるため、回避テクニックが積極的に議論されている。

### 4. Seedance 2.0 / HappyHorse
- HappyHorse（アリババ関連の動画生成プロジェクト）がSeedance 2.0をベンチマークで上回ったという情報が共有された。
- ただし公式の論文・パラメータ非公開で広報活動が消極的という指摘あり。
- 理由の言及は「ベンチマーク上は上回っているが、実用的な情報が少なすぎる」という懐疑的な評価。
