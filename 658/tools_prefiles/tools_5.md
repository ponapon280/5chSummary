**抽出結果（ツール関連のみ）**

- **ComfyUI（comfy）**  
  複数人で「基本ワークフローのみ覚えれば後はノードを1つずつ追加していくだけ」「総合的なベースUIとして利用」「海外情報・最新技術はComfyUIが9割以上」「A1111からComfyUIに移行した」「detailer/XYplot/hiresはA1111系に完敗」「ノードを何度も繋ぐのが面倒」「顔修復（FaceDetailer）が特に直感的でない」「ワークフローのデカさに見かけ倒し」「簡単すぎて逆に挫折する人もいる」「DAGのGUIに拒否感がある人もいる」といった声多数。  
  **選ばれている理由**：最新ノード・技術への対応が早く、柔軟にワークフローを組めるため。コード連携や自作ノードもやりやすい。

- **A1111系 / webUI（Automatic1111）**  
  「detailerとXYplotだけはA1111系の方が圧倒的に楽」「ON/OFFがワンボタンで済む」「ComfyUIより設定はともかく操作が簡単」といった比較が複数。ComfyUI移行後も一部機能でA1111に戻るor併用する人がいる。

- **SwarmUI**  
  「何も考えなくていいから楽」「ComfyUIをwebUIっぽく動かせるお手軽版」「最新ノード対応が遅れる場合がある」  
  **選ばれている理由**：ComfyUIの機能を簡単なUIで使いたい人向け。ただし上級者・最新機能重視の人はComfyUI本体を推奨する声あり。

- **Forge / reforge / neo**  
  「reforgeは使ってないが旧forgeとneoは使ってる」「未だにeasy reforge使ってる」  
  （Forge系UIの利用報告のみ）

- **カスタム環境（Python + Docker + Next.js）**  
  「ComfyUIではなくワークフロー全部をPythonコードで自作」「各ワークフローごとにDockerイメージを作成」「フロントはNext.js + Cloudflare Tunnelで認証付き公開」「スマホや安いChromebookから操作可能」「Claude Codeでワークフロー修正・Git管理」  
  **選ばれている理由**：バージョン競合を避けやすく、バージョン管理・リモート操作が快適。GUIの煩雑さをコードで解決したい人向け。

- **pixai**  
  「pixaiで日本語プロンプト対応が良かった」「pixai限定モデルが出てる」  
  （サービスとしての言及のみ）

**Web検索による参考情報**  
- SwarmUI：ComfyUIをベースにした軽量フロントエンドUI（2024-2025頃に登場）。ComfyUIのノードを簡易UIで扱えるが、一部最新カスタムノードの対応が遅れる場合がある。  
- Forge系（forge / reforge）：Stable Diffusion WebUIの派生プロジェクト。メモリ効率や速度を改善したフォークとして知られる。
