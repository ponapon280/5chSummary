# 元の本文
**生成AI関連ツール レポート**

### 1. ComfyUI（Comfy / Comfy Desktop）の変化が最も大きい
ComfyUIは複数回の抽出を通じて一貫して主流ツールとして位置づけられているが、**今回特に状況が変化**している点が目立つ。

**主な変化点**
- **Comfy Desktop（旧ComfyUI Desktop）の挙動変更**：名称変更に加え、起動・終了時のクラウド/ローカル選択が強制されるようになった。これにより「すぐ立ち上げてすぐ閉じられる」という従来の利点が失われたと不満の声が増加。ブラウザ版スタンドアロンへ移行した事例も確認された。
- **Core構造の変更**：複数インスタンス対応やスナップショット自動取得など、内部構造のアップデートが進んでいる。
- **ワークフロー構築の煩雑さに関する言及増加**：隙間制御の難しさやDAG形式への拒否感が改めて指摘されており、「基本だけ覚えれば後はノード追加で済む」という肯定的意見と「ノードを何度も繋ぐのが面倒」という否定的意見が併存。

**選ばれている理由（変化後も継続）**
- 最新ノード・技術への対応速度が速く、カスタムノードやLLM連携がしやすい。
- rgthreeのContextノードなど、必須級のカスタムノードが存在する一方で周知されていないため、初心者との乖離が拡大傾向にある。

### 2. 新たに登場・注目されたツール
前回・前々回の抽出ではほとんど言及がなかった、または重要度が上がったツールを以下にまとめる。

- **SwarmUI**  
  ComfyUIをベースにしつつ、webUIのような簡易操作を実現するフロントエンドとして新たに言及。  
  **選ばれている理由**：ComfyUIの機能を「何も考えなくていい」レベルで使いたい人向け。ただし最新ノード対応が遅れる可能性があるため、上級者は本体を推奨する声もある。

- **HeartMuLaノード**  
  音楽生成用途で登場。処理が非常に重い（2分生成に8時間規模）ため、専用ノードからの乗り換えが推奨されている。

- **Python + Docker + Next.jsによるカスタム環境**  
  ComfyUIのGUIを介さず、ワークフロー全体をコードで自作するアプローチが新たに報告された。  
  **選ばれている理由**：バージョン競合を避けやすく、リモート操作（スマホ・Chromebook対応）やGit管理がしやすい。Claude Codeとの組み合わせでメンテナンスも行っている事例あり。

### 3. 相対的に地位が低下したツール
- **Forge / reforge / ForgeNeo**  
  以前は一定の支持があったが、ComfyUIへの移行が進み、言及頻度・評価ともに低下傾向。ForgeNeoでは拡張機能の干渉による動作低下が指摘されている。

### 4. 変化が少なく省略したツール
以下のツールは前回・前々回と大きな変化がなく、選定理由もほぼ同一のため詳細を省略：
- Kohya sd-scripts（T-LoRA対応を望む声は継続）
- AIToolkit、TensorBoard、wandb、RedrayzGUI（学習監視用途）
- LECO（erase用途）
- Stability Matrix（環境構築の容易さは継続して評価）

**総評**  
今回は**ComfyUIのデスクトップ版変化**と**SwarmUI・コード自作環境**という新しい選択肢の登場が主なトピックとなっている。ComfyUI自体は依然として優勢だが、UIの使い勝手や運用形態で分化が進んでいる状況が読み取れる。
