# 元の本文
**なんJ AI画像生成スレッド 統合レポート**

### スレッド概要
本スレッド群は、主に**ComfyUI**を軸としたローカル環境でのアニメ画像・動画生成をテーマに、**Anima**（asuma Anima / WAI Anima / base-v1.0 / preview3 / Sam_ANIMAなど）モデルの運用と評価が中心。LoRA自作、動画生成ワークフロー、Ideogramとの比較、ツールのトラブルシューティングなどが活発に議論された。一部スレでは荒らし被害も発生した。

### 主なトピック（重複削除・初出中心）

**Animaモデルの評価と運用**
- base-v1.0とpreview3の性能差が繰り返し指摘され、「baseで画風LoRAが上手く作れない」声が多い一方、preview3で学習後にbaseで使用するワークアラウンドが共有された。
- 自然言語プロンプトへの対応力が高く、長いプロンプトや複雑な動作指示が通りやすい点が評価。一方で「three quarter viewなどのアングル制御は強いが、全体の絵の質が落ちた」という指摘も。
- NAIで叩き台を作ってAnimaでi2iするハイブリッド運用例も報告された。

**LoRA学習関連（T-LoRA・LECO含む）**
- T-LoRAは依然として学習が難しく、通常LoRAとの設定差が原因と見られている。
- 画風LoRAは50〜200枚程度のデータセットが主流で、loss値の目安（0.10〜0.20）とエポックごとのサンプル確認の重要性が強調された。
- LECO（概念消去）では「gigantic breasts」などをtargetとした具体例が共有され、キャプションの有無についても画風LoRAは不要、キャラLoRAは質重視で推奨という意見が分かれた。

**動画生成**
- 10erosワークフローの評価が高く、RIFE VFI v4.26移行によるフレーム補間高速化が報告された。
- Wan（EasyWan）は2.2以降更新が停滞しており、LTX派生やHunyuan Videoとの比較が進んでいる。
- 5秒程度のループ動画生成需要が高く、VRAM16GB＋80GB以上RAMが推奨されるなどハードルが高い状況が共有された。

**ツール・環境関連（初出・特徴的話題）**
- ComfyUI Desktop版で「クラウド強制選択」や終了時挙動の問題が複数報告され、ブラウザ版回帰の動きが見られた。
- Forge Neoは古い拡張機能削除で動作が改善した事例あり。
- HeartMuLa（楽曲生成）は処理が重く「2分生成に8時間予測」といった声も。
- SwarmUIはComfyUIをより手軽に扱うフロントエンドとして名前が上がった。
- マルチGPU並列生成や、Python＋Docker＋Next.jsで自作ワークフローを構築する上級者層の存在も確認された。

**その他の話題**
- Ideogramは検閲回避策（JSON Prompt Builder、1ステップスキップなど）が共有されたが、依然として厳しいとの評価。
- 実写ロリ表現や無断転載・AI生成物の扱いに関するモラル議論も散見。
- 一部スレではID:yA7PzGAEMによる大量荒らし（意味不明文字列投稿）が発生したが、通報脅しにより比較的早期に沈静化した。
