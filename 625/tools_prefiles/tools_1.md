### 生成AI関連「ツール」に関する話題抽出結果

ログから「ツール」（ComfyUI(comfy), A1111, webUI, SUPIR/nano-banana類似のUI/ノード/拡張など）に関する話題を抽出。モデル（NAI, Pony, illustrious/リアス/ill/IL, Noobai, FLUX, Wan, Qwenなど）は一切除外。ツールの選ばれている理由（例: 利便性、互換性、機能性）が明記されている場合のみ追記。

#### ComfyUI (comfyui, Comfy, comfiuiなど)
- **38**: comfiuiのバージョンアップ(0.14.1)の度にソースを手修正するのが疲れたため、チャッピーとバイブコーディングで「ファイル内の特定のキーワードに反応してソースを書き換える魔法(.py)」を作成。**理由: バージョンアップ時の手修正作業を自動化するため（無料チャッピーで高度なコーディングが可能）**。
- **48**: Comfy 0.14.1にアップしたらComfyUI-Florence2が(IMPORT FAILED)になった。
- **50**: comfyuiアプデが多いのは良いが、毎度互換性調整するのが面倒なのでしばらく上げない。**理由: アプデ頻度が高いが互換性調整が煩雑（怠い）**。
- **54**: comfyui 0.14.0からfluxもNAG対応した。
- **55**: ComfyUI 0.14.1でFlorence2は動いたが、LayerStyle AdvanceのFlorence2が「モデルがねーよ」エラー。
- **56**: ComfyUI-Florence2のIMPORT FAILED（transformersのHybridCacheインポートエラー）。**理由: キャッシュ仕様変更による互換性問題**。
- **57**: prompt all in oneの設定（WebUIの設定欄ではなくアイコンの歯車>筆）。NEOのfork版で設定項目がない。
- **58**: ComfyUI 0.14.1前後からSageAttention使用で生成が遅くなる気がする。
- **60**: torch compileは初回時間かかるが速くならず外した。gefoドライバをgame用からstudioに変えたら10%早くなった。
- **61**: バージョン上げたらaimdoがエラー。ComfyUI ver13でtorchコンパイルが初回時間食う（ver12は速かった）。
- **62**: ComfyUI 0.14.xは要注意。
- **64**: 起動時のログ確認を推奨。
- **79**: torchaoのバージョンがPython3.13.12/PyTorch2.9.1+cu130に適合せず警告。適合版にするとACE-Step 1.5が動かない。LayerStyle Advance/ComfyUI-Florence2関連。
- **95**: Comufyは暴れ馬（依存関係の調整が難しい）。
- **97**: LayerStyle AdvanceじゃなくComfyUI-Florence2なら使える。pip checkでno broken。
- **99**: comfyui上でconfigフォルダ指定でqwenのウェイトをテキストエンコーダーとして対話可能に。**理由: サイズ次第で効率的**。
- **102**: WindowsUpdate[KB5074109]の影響可能性。環境固定のeasyWanで速度確認推奨。
- **123**: ComfyUIちゃんに高効率LLMノードをCoreでサポートしてほしい。
- **130**: KB5077181インストール後生成速度変化なし。自前環境で速度確認。
- **135**: comfy 0.14.2 北（アップデート情報）。
- **142**: save-metaうんたらノードがjxl使えるがWindowsビルドしんどい。結局webp。
- **168**: ComfyUIで動かすSAM3DObjectsがめっちゃ軽い。テクスチャベイクも商用利用可能。**理由: 軽量で商用利用選択肢あり**。
- **190**: 下絵用ワークフローでi2i（全体にかける）。
- **193**: sam3でキャラ/目的のものだけ差し替え（detailer類似）。
- **194**: 4060tiで誘導用下絵からリアス完成まで2回目15秒以内、sam3差し替え20-30秒。**理由: 実用的速度で高速**。
- **195**: タグ並び順の重要性（年代タグ先頭など）。
- **200**: i2i時は高速化乗せて下絵30stepを10-15stepで切り上げ。**理由: 誘導レベルで十分、色出しが良い**。
- **205**: Comfyアップデートでワイルドカード効かなくなった。
- **206**: wildcardsフォルダにyaml放置でアルファベット順読み込み障害。

#### WebUI / A1111系 (prompt all in oneなど)
- **44**: prompt all in oneでプロンプト先頭にスペース入れる設定。NEO fork版で設定項目なし。
- **57**: WebUI設定>プロンプトの形式>キーワードのコンボ後のスペース削除チェックを外す。

#### Kohya LoRA GUI
- **32**: kohya lora guiを使う限り教師画像は自動縮小されるので心配不要。**理由: 自動縮小で手間不要**。
- **35**: 512×768に縮小して渡していたがフラシーボかも。
- **63**: gui使ってたから問題起きてなかった。

#### その他のツール/ノード (SAM3, Florence2などComfyUI拡張含む)
- **108**: detailer（髪/瞳色変更に不向き？）、sam3のマスクは良いがインペイントで対応。
- **184**: SAM3 Text Segmentationはコンマよりandで区切る方が効きが良い気がする。

#### 画像保存/変換ツール
- **112-118,127-128,131-132,142,156**: png/jpeg/webp/jxl保存議論。としあきの「プロンプト維持してリサイズ、JPG変換スクリプト」使用。**理由: 容量軽減、プロンプト保持、圧縮劣化気にせず共有向き、手間少ない**。
- **134**: webp静止画をwebmと勘違いして待つ事故多発。

抽出対象外（モデル関連のためスキップ）: anima, リアス, ZI/QI, Wan/easyWan, noob, Queen, ill, QwenVLなど一切の言及。ツールのみに限定し、重複/文脈省略なしで完全列挙。
