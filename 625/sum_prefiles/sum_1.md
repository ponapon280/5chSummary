# なんJスレッドレポート: AI画像生成スレッド（ComfyUI/Anima中心）

## 1. スレッド概要
- **スレッドテーマ**: 主にComfyUIを活用したAI画像生成（Stable Diffusion系モデル）のTips共有、アップデート対応、モデル比較（Anima, リアス, ZI/Queen/Noobなど）。Animaモデルのプロンプト工夫やワークフローが中心。荒らし（デブニキ関連画像連投）が散見され、多少の煽り合いあり。
- **期間/規模**: ログはレス230前後（一部抜粋）。前スレからの引き継ぎで、ComfyUI 0.14.xアップデート直後と思われる（2026/02頃のタイムスタンプ）。
- **参加者傾向**: 経験者中心。初心者向けTipsも共有。Anima推し派と懐疑派の議論活発。中国モデル（ZIなど）への言及多め。
- **全体ムード**: 技術共有がメインだが、荒らし・モデル叩きで荒れ気味。ポジティブなTips交換が活発。

## 2. 主要トピックと議論ポイント

### 2.1 荒らし・迷惑行為
- **内容**: デブニキ（ぽっちゃり体型好き？）関連画像の連投荒らしが話題。Catbox/litterに迷惑、時間限定投稿で回避。金払って連投規制緩和疑惑。前のグロ荒らしとは別で「私念」説。
- **反応**: 嫌悪強い（「死ねばいい」「公害」）。ぽっちゃりWF（ワークフロー）共有が荒らし誘発？と推測。
- **影響**: スレ流され再掲多発。

### 2.2 ComfyUIアップデート関連トラブル
- **バージョン**: 0.14.1/0.14.2でエラー報告多数。
  | 問題 | 詳細 | 解決策/コメント |
  |------|------|---------------|
  | Florence2 IMPORT FAILED | HybridCache importエラー（transformers更新）。LayerStyle Advance版でモデル未検知。 | torchaoバージョン適合確認。ComfyUI-Florence2は動く場合あり。 |
  | SageAttention遅延 | 生成速度低下。WindowsUpdate（KB5074109/KB5077181）影響？ | torch compile外す、gefoドライバ変更（Game→Studioで10%向上）。easyWanで速度確認推奨。 |
  | Wildcards無効 | yamlファイルアルファベット順読み込み障害。 | フォルダ整理。 |
  | aimdoエラー, MemoryError | AnimaでVBAR allocation failed。 | 未解決多め。 |
- **対応策**: アップデート控え推奨（「暴れ馬」）。ver13/12の方が安定。

### 2.3 AnimaモデルTips（最多議論）
- **強み**: ポーズ/自然言語追従性抜群（ツイスター的指示、バックハグ/松葉崩し/百合体位可）。背景強い。画面位置指定（横ずらし/並べ）効く。i2i/下絵用最適（半stepで雑下絵→リアス仕上げ）。
- **弱み批判**: 版権キャラ弱い、ガビついた質感、データ量不足で発展性疑問（「過大評価」「ゴミ」派）。名前が検索しにくい。
- **プロンプト工夫**:
  | カテゴリ | Tips |
  |----------|------|
  | 効果音 | Animayumeで絵師タグ+効果音多用。「in Japanese」追加（英語擬音→日本語）。ネガ: English/Chinese/Korean text。最後に効果音。 |
  | 重ね着 | jacket over hoodieタグ（Danbooru 2.1k）。自然言語キャプション。LoRA無しでも可。 |
  | 体位/ポーズ | 詳細英語記述（e.g., "matsuba kuzushi position, leg up, straddling"）。タグオーダー: [quality][1girl][char][series][artist][general]。 |
  | 特殊 | 乳搾り: "Milking cups with tubes" or "areolae grip"。パンスト越しパンティー容易。温泉乳首沈め可。 |
  | 高速化懸念 | Lightning/DMD系追加可能性低（追従性犠牲）。30step→15stepで下絵用。 |
- **LoRA/学習**: kohya GUIでlatent512x512自動縮小。教師画像サイズ心配無用。LoRA適用時、未学習概念（墓石等）で細部潰れ。
- **作例共有**: スパロボ風chibi、reverse spitroast、正常位（横長アス比推奨、結合部見え指定NG）。

### 2.4 モデル比較（Anima vs 他）
| モデル | 強み | 弱み | コメント |
|--------|------|------|----------|
| **Anima** | ポーズ/自然言語/背景。CN無しプロンプト安定。下絵+i2i最強。 | 版権/高速版未整備。マーケット小。 | 「素ポテンシャル楽しい」「役割明確」。 |
| **リアス** | i2i/Detailer強い。LoRA多。 | ポーズ/絡み弱い。背景悩み。 | 下絵Anima+リアスi2i推奨。 |
| **ZI/Queen/Noob** | LoRA/データ量/汎用性高。アニメ調優良。 | - | 「本命」「Danbooruチューン待ち」。Animaは下絵止まり？ |
| **Illustrious** | POVハグ難。 | - | Animaで克服。 |

- **将来性**: ZIアニメ特化待ち。Anima正式版（1024x?）でLoRA/プロンプト環境強化。Comfy.org共同開発で投資安定。

### 2.5 ワークフロー/ツールTips
- **下絵フロー**: Anima雑下絵（半step）→リアスi2i（SAM3 detailer/差し替え）。4060tiで15-30秒。
- **SAM3**: Text Segmentationで"and"区切り効き良い。髪/瞳色変更はインペイント推奨。
- **保存形式**: PNG（プロンプト保持）主流。アップ時JPG/WebP（容量軽減、スクリプト活用）。JXL推し少数（エンコード遅い）。
- **その他**:
  - Prompt all in one: 設定>プロンプト形式>コンマ後スペース削除外す。
  - 音声AI（Voicebox/Qwen3-TTS）: オープンソースだが「大したことない」。
  - 動画: wan2.2+smooth+dasiwa継続。Seedanceオープンソース予定（150GB?）。

## 3. コミュニティ動向・トレンド
- **ポジティブ**: Tips豊富（プロンプト英作文化）。新要素（文字/ドット絵）試行錯誤楽しい。選択肢増加歓迎。
- **ネガティブ**: モデル分散懸念。一強時代終了。規制（Shib/Seedance IP制限）話題。
- **外部ツール**: Grok/GeminiローカルLLM検討。QwenVL/Searge LLM代替。

## 4. 結論・示唆
スレはAnimaのポテンシャルを活かした実践共有が活発で、ComfyUIユーザー向け高価値。アップデート多発でトラブル多めだが、互助精神強い。Animaは「下絵革命」として定着しそうだが、ZI/Noobのアニメ特化で覇権争い熱望。荒らし抑制と正式版待ちが今後の焦点。初心者はkohya LoRA GUI+Animaプロンプトから推奨。
